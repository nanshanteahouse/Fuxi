# 细胞类型注释

> 第 05 步：把"Cluster 0、Cluster 1……"翻译成"杆状光感受器细胞、双极细胞、Muller 胶质细胞……"——这是从计算编号到生物学命名最关键的一步。

## 我们要解决什么问题？

经过上一步的聚类，你得到了一堆编号：Cluster 0、Cluster 1……直到 Cluster N。你知道每个簇里的细胞在表达上相似——但它们是谁？

**这就是细胞类型注释（cell type annotation）要回答的问题。**

它看起来简单——找到几个标记基因，看看哪个细胞类型符合。但实际做起来，你面对的是这些挑战：

### 挑战 1：标记基因不是绝对的

Marker 基因的表达是"相对富集"而非"绝对存在"。一个完美的 Rod (杆状)光感受器标记基因 Rho（视紫红质），在 Rod 中高表达，在 Cone（锥状细胞）中也有低水平表达。如果你的数据来自视网膜周边部（Cone 很少），标不标记 Rho 可能更难判断。

而且——同一篇论文中报告的 Marker，在另一个实验条件下可能不成立。物种差异、发育阶段差异、疾病状态差异都会改变 Marker 的表达模式。

### 挑战 2：注释粒度是主观的

"T cell"、 "CD8+ T cell"、 "CD8+ naïve T cell"——这三者之间没有客观的对错之分。哪个粒度的标签"正确"，取决于你要回答的问题：

- 研究免疫全景 → "T cell" 就够了
- 研究 T 细胞分化 → 必须分到 "Naive / Effector / Memory"
- 研究 CAR-T 治疗后的免疫重编程 → 可能需要更细的 "CD8+CD45RA+CCR7+ naïve T cell"

### 挑战 3：新型细胞类型没有已知标记

如果你在研究的是一个研究较少的组织（如人类肺部的离子细胞，ionocyte），或者是一个疾病模型中出现的新细胞状态，你的知识库（KB）里可能根本没有对应的条目。

### 挑战 4：每个组织有自己的"细胞词典"

视网膜有杆细胞/锥细胞/双极细胞。肝脏有肝细胞/Kupffer 细胞/星状细胞。胰腺有 α/β/δ/PP 细胞。这些细胞类型词汇完全不同——你不可能用一套通用的 Marker 解决所有组织的注释问题。

## Fuxi 的三种注释模式

Fuxi 根据你的配置，自动选择最合适的注释模式，优先级从高到低：

### 模式 1：Unified KB 模式 — 完全的知识驱动（最强）

**适用条件：** `CFG.tissue_kb` 设置了组织知识库路径（当前只内置了视网膜知识库）。

这个模式是 Fuxi 最强大、最完整的注释路径。它的全流程：

```
分步流程:
1. Wilcoxon rank-sum → 每个簇的 Marker 基因列表
2. 超几何检验（Hypergeometric test）→ 每个簇 vs KB 中每种细胞类型，Marker 基因富集是否显著？
3. 余弦相似度（Cosine similarity）→ 簇的表达谱与 KB 中细胞类型的"轮廓"是否相似？
4. 专家规则（Expert rules）→ 硬逻辑："如果基因 A 在簇 X 中 > 基因 B 中，且 A 是 Rod 的确认 Marker → 注释为 Rod"
5. 低质量簇检测 → 如果以线粒体/核糖体基因为主，标记为 "Unknown"
6. 证据融合（5 层决策）→ 综合所有证据，给每个簇一个最终注释
7. 可选 AI 回退 → 低置信度簇交由 LLM 辅助判断
```

#### 超几何检验——为什么是它？

超几何检验的直觉：假设总基因集是 20,000 个，KB 中 "Rod" 类型的确认 Marker 是 50 个。如果簇 0 有 200 个 Marker 基因，其中 30 个碰巧在在这 50 个中——这是巧合的概率有多低？

超几何分布量化了这种随机重合的概率（p 值）。p 值越小，说明簇 0 的 Marker 与 Rod 的 Marker 高度富集，不太可能是巧合。

#### 证据融合的 5 层决策逻辑

Fuxi 的融合引擎按以下优先级合并所有证据：

| 优先级 | 证据来源 | 说明 |
|:--|:--|:--|
| 1 | 专家规则（Expert rule） | "如果 X > Y，则判定为 Z"的确定性逻辑 |
| 2 | 超几何得分 | Marker 富集的统计显著性 |
| 3 | 余弦相似度 | 表达谱层面的总体相似度 |
| 4 | 层次一致性 | 是否与同层次其他簇的注释自洽 |
| 5 | AI 建议（可选） | 低置信度簇请 LLM "看看" |

对于每个簇，融合引擎输出：
- `cell_type`：最终细胞类型
- `confidence`：`high / medium / low / unknown`
- `method`：最终决策依赖的证据层（`expert_rule`、`marker_scoring`、`marker_scoring+ai`）
- `explanation`：可供溯源的解释

#### 知识库的更新机制

Fuxi 的 KB 不是死的。每次 Unified KB 注释完成后，Fuxi 会生成一份质量报告（`05_annotation_quality.json`），告诉你：
- Marker 验证通过率（PASS rate）
- 有多少簇被规则注释，多少被打分注释
- AI 建议与 KB 建议的一致性

如果 PASS rate < 10%，Fuxi 提示可能 KB 对这个数据集覆盖不足——换用更宽松的 strictness 设置或 score_genes 回退模式。

### 模式 2：AI 模式 — LLM 驱动的智能注释

**适用条件：** `CFG.ai.enabled` 和 `CFG.ai.ai_annotation` 都设置为 True，但没有设置 `tissue_kb`。

在这个模式中，Fuxi 把前几步计算的 Marker 基因输送给一个 LLM（GPT-4、DeepSeek 等），并提示模型进行推理：

提示词模板包含的信息：
- 组织（tissue）："视网膜"、"外周血"、"胰腺"等
- 物种（species）："人"、"小鼠"、"斑马鱼"等
- 每个簇的前 30 个上调 Marker 基因（含 log2 fold-change 和 p 值）
- 发育阶段信息（如果有）

LLM 需要返回每簇的 JSON 结构：
```json
{
  "0": {
    "cell_type": "Rod Photoreceptor",
    "subtype": "Mature Rod",
    "state": "N/A",
    "confidence": "high",
    "reasoning": "Top markers include Rho, Pde6b, Nr2e3. Rho is the canonical rod opsin..."
  }
}
```

AI 模式的优势是它能"理解"——如果某个簇的 Marker 基因在文献中曾被关联到多种细胞类型，LLM 能基于你的组织 + 物种信息做权衡判断。缺点是需要 API 调用、可能有延迟、对异常值敏感。

> **Fuxi 的自适应 max_tokens：** 簇数 * 300 token + 基数，夹在 4096~32768 之间。如果你的数据有 30 个簇，标注一个簇的完整 JSON 大约需要 300~500 token。

### 模式 3：Score_genes 模式 — 简单直观的回退

**适用条件：** 以上两种模式不可用或失败时。作为兜底方案。

逻辑非常简单：
1. 用户定义 `marker_dict`，如 `{'Rod': ['Rho', 'Pde6b'], 'Cone': ['Gnat2', 'Arr3']}`
2. 对每种候选细胞类型，`sc.tl.score_genes()` 计算每个细胞的得分
3. 每个簇内取平均分，得分最高的类型即为该簇的注释
4. 置信度 = 最高分 - 次高分

这本质上是"最像哪个已知类型就标哪个"——简单、透明、快速。但它不擅长处理高度相似的类型，也不擅长处理 KB 中没有的类型。

## 低质量簇的处理

这是一个重要的实践细节。Fuxi 的 Unified KB 模式内置了低质量簇检测。通过检查每个簇的 Marker 基因列表，如果一个簇的 top markers 一半以上是线粒体基因（MT-）或核糖体基因（RPS/RPL），它会被判定为低质量簇。

对于低质量簇，Fuxi 的融合引擎强制标注为 "Unknown"，即使统计指标指向某种细胞类型。

为什么？因为一个以线粒体基因为 top markers 的簇，生物学上几乎不可能对应任何真实的细胞类型——它是死细胞或受损细胞的"残余物"。给它一个错误的细胞类型名称（如 "Rod"）比标记为 "Unknown" 更具误导性。

## 注释标准化器（Annotation Standardizer）

当你使用 AI 模式或 Unified KB 模式时，Fuxi 可选地应用**注释标准化器（StandardOntology）**。

它解决的问题：LLM 可能标注为 "retinal rod cell"、另一个数据集标为 "Rod photoreceptor"、第三个数据集标为 "rods"——三个名字说的是同一个东西。标准化器将它们映射到统一的术语表。

标准化器输出三列：
- `cell_type`：最终使用的显示名称
- `cell_type_std`：标准术语表中的名称
- `cell_type_raw`：LLM/KB 原始输出

这样你既能得到标准化的名称用于跨数据比较，又能追溯到原始输出。

## 我怎么看结果？

### 日志输出（Unified KB 模式示例）

```
Unified KB mode enabled — tissue_kb='retina'
Loaded KB: retina (8 cell types, 12 rules)
  Cluster 0 → Rod Photoreceptor (conf=high, method=expert_rule)
  Cluster 1 → Cone Photoreceptor (conf=high, method=marker_scoring)
  Cluster 2 → Müller Glia (conf=high, method=expert_rule)
  Cluster 3 → Unknown (conf=unknown, method=unknown)     ← 低质量簇
  Cluster 4 → Bipolar Cell (conf=medium, method=marker_scoring)
  Cluster 5 → Rod Photoreceptor (conf=high, method=marker_scoring+ai)
  ...

```

关注以下几点：

- **高质量注释（conf=high）**：信心很足的标注，通常对应"rules matched"或"strong hypergeometric enrichment"
- **中低置信度（conf=medium/low）**：可能需要手动检查
- **Unknown 簇**：通常就是低质量簇。如果某个看起来有生物学意义的簇被标为 Unknown，需要进一步调查
- **pass_rate**：总体质量指标，建议 >50%。如果很低，考虑放宽 strictness 或切换为 AI 模式

### UMAP 注释图

`_05_celltype_unified.pdf` / `_05_celltype_ai.pdf` 是将 UMAP 按注释的 cell_type 着色的图。检查：
- 相同类型的细胞是否形成连贯的区域？
- 有没有某个类型的细胞散布在多个不连续的区域？（可能注释有误或分层聚类时需要再细分）
- Unknown 簇是否在图中边缘位置？（低质量细胞通常分布在 UMAP 边缘）

`_05_confidence_unified.pdf` 按置信度着色——如果大部分区域是红色（low/unknown），说明注释整体质量不高。

## 常见误区

**误区 1：**"Fuxi 注释完就不用再检查了。"
最大的误区。Fuxi 的注释是一个**高质量的初始假设**，不是最终真理。特别是：
- 对于研究较少的组织，KB 可能覆盖不全
- AI 注释犯过一次"幻觉"（hallucination）——把完全错误的细胞类型描述得头头是道
- **总是用已知的 Marker 基因手动验证几个关键的簇**

**误区 2：**"Score_genes 模式简单直接，效果应该差不多。"
Score_genes 模式只做"取最高分"——它对相似的细胞类型（如 CD4+ vs CD8+ T 细胞）区分力很弱。Unified KB 模式或 AI 模式通常做得更好。

**误区 3：**"我研究的是小鼠肝脏，Fuxi 怎么没有肝脏的 KB？"
KB 需要人工从文献中整理，目前只内置了视网膜的 KB。如果你用的是其他组织，推荐使用 AI 模式或 Score_genes 模式。后续我们会逐步扩展更多组织的 KB。

**误区 4：**"所有簇都应该有一个有意义的细胞类型名称。"
不对。数据中总是存在一些无法被注释的簇——低质量细胞、过渡性细胞状态（如正在分裂的细胞）、或者极罕见的细胞类型。把这些簇标记为 "Unknown" 是一个诚实的做法。**给一个簇强行分配一个错误的类型，比留它为 Unknown 更糟糕。**

**误区 5：**"注释可以跨数据集直接比较。"
两个实验室用 Fuxi 标注同一组织，可能得到不同的注释粒度。一个用 Resolution 0.5（分到 8 个簇），另一个用 Resolution 1.5（分到 20 个簇）。细胞类型名称也可能不同（"Rod Photoreceptor" vs "Rod cell" vs "Rods"）。这就是**注释标准化器的价值**——但标准化不是自动魔法，它需要你建立一个统一的命名约定。

## 小结

细胞类型注释是单细胞分析中**最需要生物学知识**的步骤。Fuxi 的三种模式覆盖了从纯知识驱动（Unified KB）、到 AI 辅助（AI 模式）、到简单回退（Score_genes）的全谱系需求。

它们的核心思想是一致的：
1. 找到每个簇的特征基因（Marker）
2. 用这些 Marker 与已知细胞类型的特征匹配
3. 匹配的证据越强，置信度越高
4. 匹配不上的，诚实地标为 Unknown
