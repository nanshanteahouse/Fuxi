# Subcluster 亚群分析

> 细胞类型注释是在画地图上的大洲——但一个大洲内部可能还有山脉、平原、河谷。Subcluster 就是放大镜，让你看到大洲内部的精细结构。

## 我们为什么要做 Subcluster？

在完成主聚类和注释之后（[Step 05](09_细胞类型注释.md)），你手上有一张细胞图谱：哪些细胞是感光细胞（photoreceptor）、哪些是双极细胞（bipolar cell）、哪些是 Müller 胶质细胞。每个"大类"都被标上了标签。

但问题是——**这些大类本身可能是"混合群体"（heterogeneous population）**。

拿视网膜的双极细胞（Bipolar Cell）来举例。在视网膜的经典分类里，"双极细胞"本身包含至少 10 个以上的亚型：ON-Bipolar、OFF-Bipolar、Rod Bipolar……每一种亚型表达不同的基因、连接不同的下游神经元、对光信号做出不同的反应。但在你的 UMAP 上，所有这些亚型可能被压缩在一个大 cluster 里，看起来像一个均匀的群体。

这是因为 Step 04 的主聚类（main clustering）所选择的 Leiden 分辨率是**为了把所有细胞类型区分开而优化的**——它的目标是让你看到"视网膜有感光细胞、双极细胞、Müller 胶质细胞……"而不是让你看到"双极细胞内部有 10 种亚型"。如果把分辨率调高到能分出所有亚型的程度，你会得到几十个 cluster，反而看不清大格局。

**Subcluster 的策略是先看大格局，再聚焦到某一类，在局部重新聚类。**

## Subcluster 的原理

Subcluster 的逻辑非常直观：

1. **抽取子集：** 把某一类细胞（比如所有标注为"Bipolar Cell"的细胞）从完整数据集中提取出来
2. **重新降维 + 聚类：** 在这个子集上重新运行 PCA → UMAP → Leiden 聚类。注意，这一步的子聚类分辨率可以（而且应该）比主聚类高——因为你现在只关注一个细胞类型的内部结构，没有"把大类搞混"的风险
3. **重新注释亚型：** 对 subcluster 产生的子类重新做标记基因分析和注释

这个过程等价于：先拍一张城市地图（主聚类），然后把市中心放大（subcluster），再看市中心的每条街道（亚型注释）。

### 什么时候做 Subcluster？

**推荐场景：**

- **大类细胞数量>500：** 统计学上，一个群体有足够多的细胞，Leiden 才能找到有意义的子结构。少于 200 个细胞的群体做 subcluster，结果可能不可靠（统计学功效不足，容易被少数离群细胞带偏）
- **已知该细胞类型具有功能亚型：** 这不是必须的，但如果你知道文献中报道了某个细胞类型的多种亚型，subcluster 是验证它们是否在你的数据中出现的最直接方法
- **大类 cluster 的 UMAP 内部有明显"驼峰"或"亚结构"：** 有时你在大 UMAP 上就能看到某个 cluster 内部有"断崖"或者"两个密度中心"，这强烈暗示有亚群存在

**不推荐的场景：**

- **细胞数<200：** 信号会被噪音淹没。尤其注意——doublet（[Step 01 双细胞检测](05_双细胞检测.md)）在少数细胞中占比太高，会让 subcluster 结果不可解释
- **标签本身就是"垃圾"类：** 有些 cluster 在注释时被你标注为"low quality"或"unassigned"。不要对这类细胞做 subcluster ——它们只是你没测好的细胞，不是有生物学意义的亚群
- **没有生物学上的理由去期待亚型：** Subcluster 是"科学驱动的望远镜"，不是"数据分析的网红滤镜"。如果你没有理由认为这群细胞应该被细分，那就不要为了做而做

## Fuxi 是怎么做的？

Fuxi 的 Step 06 subcluster 设计为**按需运行**——它不会自动对所有细胞类型做 subcluster，因为那样既浪费时间（可能对很多不需要的类也做了），又增加了 10 倍以上的输出文件需要你去检查。

使用方式是通过 `--cell-type` 参数指定要 subcluster 的细胞类型：

```bash
python core/run_pipeline.py --modality rna --step 6 --cell-type "Müller Glia" --config ...
```

Fuxi 内部会：

1. **从 anndata 中筛选**出当前 cell type 的所有细胞
2. **重新运行 PCA（高维主成分分析：principal component analysis）：** 使用**当前子集内部**的高变基因（highly variable genes），而不是全数据集的 HVG。因为一个亚型内部的标记基因可能是全数据集范围里并不"高变"的基因
3. **运行 UMAP + Leiden 聚类：** subcluster 的分辨率由 `CFG.subcluster_resolution` 控制（默认比主聚类分辨率高；也可自行设置）
4. **自动计算 subcluster 的标记基因：** 使用 Step 07 相同的标记基因检测逻辑，但比较范围缩小到了该 cell type 内部
5. **保存 subcluster 结果：** 在 `outputs/` 下生成以 cell type 命名的子目录，包含 UMAP 图、标记基因表格和新的 anndata 文件

> 具体参数配置见 [Pipeline 指南](../pipeline_guide_zh-CN.md#step-06-subcluster)

### 命名约定

Subcluster 的命名遵循"大类名.亚类编号"的格式，例如：

- `Müller Glia.0`：Müller 胶质细胞的第一个亚群
- `Müller Glia.1`：Müller 胶质细胞的第二个亚群

这样你在后续分析（标记基因、轨迹、富集）中可以轻松追溯到母群体。

## 我怎么看结果？

Subcluster 的输出主要包含三部分：

### 1. Subcluster UMAP 图

这是最关键的一张图。你会看到原来的"Bipolar Cell"在 subcluster UMAP 上分裂成 5-6 个不重叠的群体。

**重点看：**

- **"桥接"细胞是否存在：** 如果 subcluster 之间没有清晰的边界，而是有一串连续的细胞——这可能意味着你的 Leiden 分辨率太高了，把连续渐变的状态（developmental continuum）强行切成了离散的 cluster
- **是否有极小 cluster（<20 细胞）：** 这种极小 cluster 大概率是"离群值集群"——几个异常细胞因为共享一些罕见的（可能是噪音）基因特征而被聚类在一起。这类 cluster 通常没有生物学意义，除非你验证它们有独特的标记基因

### 2. 亚群标记基因表

Fuxi 会输出每个 subcluster 相对于同细胞类型内其他 subcluster 的标记基因。

**重点看：**

- 每个亚群是否有已知的标记基因高表达？例如，Rod Bipolar 应该表达 GRM6，OFF-Bipolar 应该表达 VSX1
- 如果两个亚群的标记基因表高度相似（top 10 基因几乎相同）——你可能 over-cluster 了，它们应该合并

### 3. 亚型注释建议

Fuxi 会尝试根据标记基因自动建议亚型名称（如果你的配置启用 AI 注释）。但 subcluster 的生物学验证很大程度上需要你手动核对文献。

## 常见误区

**"Subcluster 的分辨率越高越好"**——错误。高分辨率会刻出更多的假阳性 cluster。一个简单的检查方法：把 subcluster 的标记基因画回到主 UMAP 上看——如果一个"亚群的标记基因"在主 UMAP 上散落在好几个大类中，那这个"亚群"可能只是你在局部数据中过度拟合出来的假结构。

**"Subcluster 结果中的每个亚群都有生物学意义"**——不一定。有些 subcluster 是同一种细胞状态在不同细胞周期阶段的差异（一个在 G2/M 期，一个在 G1 期）。这些是"细胞周期亚群"，不是"功能亚群"。如果你的实验问题是关于细胞类型的，你可能需要排除细胞周期基因的影响。

**"做完 subcluster 就可以直接进入下游分析"**——注意！如果你给亚群分配了新标签（如 "Bipolar.0" 改为 "Rod Bipolar"），你需要更新 anndata 中的注释列。Fuxi 默认的 subcluster 命名不会自动替换主注释——这是有意设计的：让你有机会手动审核后再决定是否"升级"某个亚组的注释。

**"所有大细胞类型都应该做 subcluster"**——不应该。Subcluster 要回答特定的生物学问题，而不是数据分析的"标准动作"。做之前问自己一句："如果这群细胞不能细分，我的生物学故事还成立吗？"——如果不成立，就做；如果故事不变，不做。

## 小结

- Subcluster 是对已注释的大类细胞类型进行"局部放大"——在子集内重新降维和聚类，找到功能亚型
- **适用条件：** 细胞数 > 500、有理由期待亚型存在、有生物学问题驱动
- **不适用：** 细胞数 < 200、标签为低质量类、没有生物学理由
- Subcluster 的分辨率需要平衡——太高会过拟合（产生假阳性小 cluster），太低会遗漏真实亚型
- Subcluster 结果是"建议"，不是"定论"——你需要用标记基因和文献验证每个亚群的生物学身份
