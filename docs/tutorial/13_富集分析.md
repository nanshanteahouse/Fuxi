# 富集分析

> 你有 200 个基因是"感光细胞的特异性标记"。很好。但 200 个基因的列表不是一个故事——你需要知道它们在细胞里一起做什么。富集分析把基因列表翻译成生物学功能。

## 我们要解决什么问题？

做完[标记基因与差异表达分析](11_标记基因与差异表达.md)，你手上有一长串基因——比如"感光细胞相对于其他细胞上调的 300 个基因"。一个典型的列表如下：

| 基因 | logFC | p_val_adj |
|------|-------|-----------|
| RHO | 4.2 | 10⁻⁵⁰ |
| CRX | 3.8 | 10⁻⁴⁰ |
| GNB3 | 2.9 | 10⁻³⁰ |
| … | … | … |
| 总共 300 行 | | |

问题来了：**这 300 个基因到底在做什么？**

你可以一个一个读文献，但效率太低。而且有些基因你甚至不知道它的功能。

富集分析（enrichment analysis）做的就是这个翻译：**"你的基因列表 VS 已知的功能基因集"**。它的核心问题是：

> 在你的基因列表中，属于"光转导通路"的基因有 30 个。但如果你随机挑 300 个基因，也会有大约 3 个碰巧属于光转导通路。30 > 3，但 30 比 3 "富集"了多少倍？这个富集程度是统计学上显著的吗？

**富集分析回答的正是这个问题。**

## 两种主要方法

### ORA（Over-Representation Analysis，过度代表分析）

ORA 是最经典、最直观的方法。

**怎么做：**

1. 取你的基因列表中的"top N"个（默认为 200 个标记基因）
2. 对每个通路（pathway），计数你的列表中有多少个基因落入该通路
3. 用**超几何检验（hypergeometric test）** 问：这个计数是否显著大于随机期望？

**一句话直觉：** "你从装满红色和蓝色球的篮子里抓一把球。抓到的 15 个球中有 10 个红色。如果篮子里红色球的比例只有 10%，你抓出 10/15 红色这件事非常不可能——所以红色球被你偏好性地富集了。通路分析的逻辑完全一样——你的基因列表相当于"抓出来的球"，通路相当于"红色球"。"

**ORA 的优点：**
- 简单、易懂、计算快
- 非常适合强信号的数据（比如标记基因列表——差异非常大）
- 结果容易可视化（气泡图、条形图）

**ORA 的缺点：**
- 需要你手动选择一个"截断值（cutoff）"——top 200 基因 vs top 500 基因可能给出不同的富集结果
- 忽略弱信号——有些通路中的基因虽然整体上调，但每个基因的倍数变化不大，它们可能在 top 100 之外被截断排除

### GSEA（Gene Set Enrichment Analysis，基因集富集分析）

GSEA 是 ORA 的"升级版"，它不要求你预先截断基因列表。

**怎么做：**

1. 把你所有基因按"表达倍数变化"从大到小排序（不截断——全部用上）
2. 对每个通路，沿着这个排序列表"走路"：遇到通路中的基因就往上走一步（加权重），不在通路中就往下走一步
3. 计算这条"行走路径"的峰高（enrichment score）——如果通路中的基因集中在排序列表的顶部，峰很高；如果它们均匀分布，行走路径平缓几乎为 0

**一句话直觉：** "你的基因排列在一根绳上（从最上调到最下调）。现在拿一个通路的所有基因当"图钉"，看它们在绳子上的位置是集中在大调的一端（富集）、集中在大调里（抑制）、还是均匀散落（无富集）。"

**GSEA 的优点：**
- 不需要人为截断——所有基因都参与分析
- 能捕捉到"弱但一致"的信号——比如一个通路中 80% 的基因都轻微上调（logFC=0.5），单个基因不显著，但整个通路协调上调这种模式是有意义的

**GSEA 的缺点：**
- 计算更复杂、更慢
- 结果解释对初学者不太直观（enrichment score 是正还是负？NES 大于多少算显著？）
- 对离群值敏感（一个基因的极高倍数变化可以拉动整条行走路径的峰）

### 什么时候用 ORA，什么时候用 GSEA？

| 场景 | 推荐方法 | 原因 |
|------|---------|------|
| 你做的是标记基因分析（一 vs 全体） | ORA | 标记基因通常信号非常强烈（logFC 大、p 值小），top N 基因足以捕获主要信号 |
| 你做的是条件间 DE（条件 A vs B） | GSEA | 条件间的变化可能比较微妙（不多但仍有一致协调性），ORA 的截断可能漏掉弱信号 |
| 你只有少量基因（<50） | ORA | GSEA 需要较长的排序列表来稳定计数 |
| 数据有多个时间点 | 两个都用 | ORA 帮你定位强信号通路，GSEA 帮你发现跨时间点的细微变化 |

Fuxi 默认两种方法都做，方便你对比验证。

## Fuxi 是怎么做的？

Fuxi 的富集分析流程分为两层：

### 第一层：基因本体论（GO）富集

**GO（Gene Ontology，基因本体论）** 是一个庞大的分类系统，按三个维度给基因打标签：

- **Biological Process（生物学过程）：** 基因参与什么"任务"（如"光转导"、"细胞周期"、"DNA 修复"）
- **Molecular Function（分子功能）：** 基因本身有什么"技能"（如"转录因子活性"、"G 蛋白偶联受体活性"）
- **Cellular Component（细胞组分）：** 基因产物在哪里"工作"（如"突触膜"、"线粒体"、"细胞核"）

Fuxi 默认对**生物学过程（Biological Process）** 做富集。这是最有洞察力的维度——它会告诉你"感光细胞在高做什么"而不是"感光细胞的零件在哪里"（那是 Cellular Component 回答的问题）。

### 第二层：KEGG 通路富集

**KEGG（Kyoto Encyclopedia of Genes and Genomes，京都基因与基因组百科全书）** 是一个通路数据库——它描述的是"基因 A → 蛋白质 A → 催化反应 1 → 激活下游通路 → ……"这种串接关系。

GO 给出的是"哪些任务正在被做"，KEGG 给出的是"这些任务之间的联系是怎样的"——比如"Phototransduction (光转导)"通路不仅列出了光转导涉及的基因，还展示了它们之间的激活/抑制关系。

### Pipeline 在做什么

在 Fuxi 中，富集分析的输入来自上一层的输出：
- **Layer 1（标记基因）** 的 top 基因 → ORA
- **Layer 2（配对 DE）** 的全部基因 → GSEA

Fuxi 同时对所有细胞类型运行富集分析——所以你会得到一份汇总表格：每种细胞类型、每个通路、ORA 和 GSEA 的 p 值和富集分数。

> 具体参数配置见 [Pipeline 指南](../pipeline_guide_zh-CN.md#step-09-富集分析)

## 我怎么看结果？

### 气泡图（Bubble chart / Dotplot for enrichment）

Fuxi 会为每个细胞类型生成一个富集分析气泡图。这是一张信息密度非常高的图：

- **X 轴：** 富集比（enrichment ratio）或 NES（Normalized Enrichment Score，归一化富集分数）——越靠右，富集越强
- **Y 轴：** 通路名称
- **气泡大小：** 交集中的基因数（你的基因列表中有多少基因落在这个通路里）
- **气泡颜色：** p 值（或校正后的 p 值）——通常是蓝色到红色渐变，红色表示更显著

**阅读顺序（三步法）：**

1. **先看颜色：** 只关注红色或深红色的气泡——这些通路统计上显著（p < 0.05）。浅蓝色或灰色的气泡通常不值得深究
2. **再看位置：** 在"红泡"中，哪个最靠右（富集分数最高）？这个通路可能是你细胞类型中最核心的生物学功能
3. **最后看大小：** 大的气泡 = 很多基因的富集，通常比小的气泡更可信。但如果一个小气泡又红又靠右（强富集 + 显著 + 少量基因），它可能代表一个高度特异性的功能

**一个示例：** 感光细胞的气泡图显示"Phototransduction（光转导）"通路是一个大红泡、靠右、大小中等。这时候你的结论是："感光细胞确实在做光转导——和已知生物学一致，这是一个验证性的结果，不是新发现。"

### 重点看什么（而不是什么）

**应该关注：**
- 与细胞类型生物学身份一致的通路：感光细胞→光转导、Müller 胶质→谷氨酸代谢、小胶质细胞→免疫响应
- 意料之外但高度显著的富集：如果你的 B 细胞 cluster 富集了"T 细胞活化"途径——这可能需要小心查证（可能是 doublet 或注释错误）
- 多个相关通路同时富集（如 "Rhodopsin signaling" + "Visual perception" + "Response to light stimulus" 在感光细胞中同时出现），验证你的注释

**不应该过度关注：**
- p 值非常小但富集比接近 1 的通路（统计功效来自细胞数量大，不是生物学信号强）
- 只有一个基因落入的通路（"富集"了一个基因——等价于"这个通路里有一个基因是标记基因"，这不是通路水平的富集）
- 名称模糊的通路如 "metabolic process"、"cellular process"——这些泛通路几乎在所有细胞类型中都会富集，信息量接近 0

### 排除"管家基因"干扰

一些通路（如"线粒体翻译"、"核糖体组装"、"RNA 剪接"）在所有细胞类型中广泛富集，因为它们对应的是**所有细胞都在做的基础过程**。这不代表生物学上的新发现，也不适合用于区分细胞类型。

Fuxi 不做自动过滤——你需要自己留意这些"管家通路"（housekeeping pathways）的干扰。

## 常见误区

**"富集分析告诉我这个通路在感光细胞中被激活了"**——不，富集分析只告诉你"感光细胞的特异性基因列表中，这个通路的基因比随机期望更多"。这**不等于"通路在感光细胞中被激活了"**。"激活"意味着通路的功能状态发生了变化（磷酸化、蛋白转运等），富集分析没有测量任何功能状态。它只测量的基因集重叠的统计学显著性。

**"更小的 p 值 = 更重要的通路"**——不！p 值受基因集大小的影响非常大。一个通路有 500 个基因和一个通路有 5 个基因，即使真实富集程度相同，500 基因通路的 p 值也会更小（因为大通路更容易捕获你的基因）。所以富集排名时，应该结合富集比（enrichment ratio）和 p 值一起看。

**"GSEA 一定比 ORA 好"**——不。GSEA 和 ORA 适合不同的问题。对于标记基因（强信号，你的数据有一个明确的"top gene cut"），ORA 更合适，因为它只使用 top 基因，不会把低表达的噪声基因拖进来。GSEA 更适合条件间比较（弱但一致信号的两个条件），它利用整个基因谱的信息。

**"GO 富集的结果就能发论文"**——不能。GO/KEGG 数据库基于文献积累，你看到的结果是**已知**功能的总结，不是你的新发现。富集分析的价值在于：帮你聚焦到几个最有希望的通路假设，然后用实验验证（qPCR、Western blot、功能实验）。富集分析是生成假设的工具，不是验证假设的工具。

**"GO 和 KEGG 已经足够"**——不够。GO 和 KEGG 偏向经典生物学过程。对于罕见疾病、非模式生物、新技术（如空间转录组、单细胞多组学），它们的覆盖可能不足。如果有特定的专业数据库（如"Phototransduction genes"类库），用自定义基因集比通用数据库效果好得多。

## 小结

- 富集分析回答"我的基因列表在做什么"——把基因名翻译成生物学功能
- **ORA** 基于 top N 基因做超几何检验：信号强、直觉直观，适用于标记基因
- **GSEA** 基于全基因排序列表：不丢弱信号，适用于条件间比较
- 气泡图阅读顺序：颜色（p 值）→ 位置（富集比/NES）→ 大小（基因数）
- 注意排除管家通路、注意大通路的人为低估 p 值、注意富集分析是"生成假设，不是验证假设"
- GO 和 KEGG 只能告诉你已知的功能——如果你的细胞类型做了前人不知道的事情，富集分析不会有结果
