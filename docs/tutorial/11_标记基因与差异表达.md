# 标记基因与差异表达

> 标记基因（marker gene）回答"这个细胞类型是什么"，差异表达基因（DE gene）回答"这个细胞类型在不同条件下发生了什么变化"。很多人混用这两个概念，但它们是两件完全不同的事。

## 标记基因和差异表达基因到底有什么区别？

假设你有一群视网膜细胞，你把它们分成了感光细胞、双极细胞和 Müller 胶质细胞。现在你有两个不同的问题：

### 问题一：这个感光细胞 cluster 为什么被叫做"感光细胞"？

你要找出那些**在这个 cluster 里表达量显著高于所有其他 cluster 的基因**。如果 RHO（视紫红质基因）在所有其他 cluster 里几乎不表达，但在这个 cluster 里高表达，RHO 就是这个 cluster 的**标记基因（marker gene）**。

标记基因的核心是**"一 vs 全体"（one-vs-rest）**。每个 cluster 和所有其他 cluster 比较，找出能把它从所有细胞中区分出来的基因。

**标记基因的用途：**
- 验证你的注释是否正确（你标注为"感光细胞"的 cluster 是否真的表达 RHO？）
- 发现新的细胞亚型（某个 subcluster 标记了一个你之前不知道的基因组合）
- 为后续的注释提供线索（特别是当你使用[AI 注释](09_细胞类型注释.md)时，标记基因是驱动 AI 判断的主要证据）

### 问题二：同一个细胞类型在两个条件下有什么不同？

现在你比较的是**同一个条件的两组样本**——比如"正常小鼠的感光细胞 vs 退化模型小鼠的感光细胞"。你想知道：在疾病条件下，感光细胞里的哪些基因发生了变化？

你要做的是**"配对比较"（pairwise comparison）**：同样是感光细胞，条件 A vs 条件 B，哪个基因上调或下调了。

**差异表达（differential expression，DE）的用途：**
- 发现疾病的分子机制（哪些通路在疾病中被激活或抑制）
- 比较两个发育阶段之间的转录组变化
- 寻找药物处理前后的响应基因

### 表格对比：标记基因 vs 差异表达

| | 标记基因（Marker Gene） | 差异表达（DE） |
|---|---|---|
| **比较对象** | 一个 cluster vs 所有其他 cluster | 同类型细胞，条件 A vs 条件 B |
| **回答的问题** | "这个细胞类型是什么？" | "这个细胞类型在不同条件下有什么变化？" |
| **结果解读** | 鉴定细胞身份 | 发现生物学差异 |
| **通常需要** | 一个分组信息（cluster 标签） | 两个分组信息（cluster 标签 + 条件标签） |

> 很多人用一个方法（Wilcoxon rank-sum test）同时做标记基因和差异表达——这正是最令人困惑的地方。方法都一样，但**解释的方式**完全不同。

## 为什么用 Wilcoxon 秩和检验？

在单细胞中，标记基因和 DE 检测最常用的统计方法是 **Wilcoxon 秩和检验（Wilcoxon rank-sum test，也称 Mann-Whitney U test）**。

它的直觉是：**不关心基因的具体表达量是多少，只关心"如果把两组细胞的表达值混在一起排序，这个组的排名是不是整体更高或更低"。**

为什么不用 t 检验？

- t 检验假设数据服从正态分布（钟形曲线）。但 scRNA-seq 的计数数据是高度偏态的（大部分是 0，少数是非零值）——完全不满足正态分布假设
- Wilcoxon 是非参数检验（non-parametric test），它不对数据的分布做任何假设。它只看"排名"，不看"数值"
- Wilcoxon 对离群值（outlier）非常稳健——如果一个细胞中某个基因计数异常高（比如 UMI 计数 5000 vs 同一类型其他细胞只有 10），t 检验会被这个极端值拉偏，但 Wilcoxon 基本不受影响

**一句话：Wilcoxon 把数值转换成排名后再做比较。排名不受极端值影响，也不依赖正态分布假设。**

## Fuxi 的三层 DE 策略

大多数管道只做一件事——计算标记基因。Fuxi 针对不同生物学问题设计了**三层策略**：

### 第一层：标记基因（Marker Genes）

每个 cluster 与所有其他细胞做 one-vs-rest 比较。

```
感光细胞 → 比较 → 所有其他细胞（双极细胞 + Müller 胶质 + ...）
双极细胞 → 比较 → 所有其他细胞（感光细胞 + Müller 胶质 + ...）
```

生成一个表格，每行是一个"cluster + 基因"对，包含 log2 倍数变化（log2 fold change）、校正后的 p 值、以及在两组中的表达百分比（pct.1 / pct.2）。

**每一层都能独立运行**——如果数据没有发育时间点（只有一种条件），Fuxi 只做第一层，然后跳过第二、三层。

### 第二层：配对 DE（Pairwise Stage DE）

只有在你的数据有多个发育时间点（developmental stages）或者实验条件时才运行。例如，你有 E14、E16、P0、P7、Adult 五个发育阶段的视网膜数据。

此时，Fuxi 对**每个细胞类型**在**相邻时间点**之间做两两比较：

```
感光细胞：E14 vs E16 → 哪些基因上调？
感光细胞：E16 vs P0  → 哪些基因上调？
...
双极细胞：E14 vs E16 → 哪些基因上调？
```

这种设计的目的是追踪"每个细胞类型随发育时间的变化轨迹"。

### 第三层：时间趋势基因（Temporal Trend Genes）

这一步需要**至少三个时间点**。Fuxi 将发育阶段编码为有序变量（E14=0, E16=1, P0=2, P7=3, Adult=4），然后对每个基因在每个细胞类型中拟合"表达量随时间的变化趋势"：是单调递增？单调递减？还是先高后低（峰型）？

**输出的基因有三类：**
- **上调趋势基因（up-trending）：** 表达量随发育逐渐升高。通常是"成熟标记"——细胞在成熟过程中"开启"的基因
- **下调趋势基因（down-trending）：** 表达量随发育逐渐降低。通常是"前体标记"——细胞在未成熟时表达，成熟后关闭的基因
- **峰型基因（peak）：** 某一个发育阶段短暂高表达。这种基因往往参与关键的"开关事件"——比如决定一个细胞是变成感光细胞还是双极细胞的命运决定基因

第三层的设计灵感来自于发育生物学家最常问的"wishbone"问题："在这个细胞类型的分化过程中，基因 A 什么时候开、什么时候关？"

## 我怎么看结果？

### 怎么看标记基因表

标记基因的 CSV 文件格式如下：

| gene | cluster | logfoldchanges | pvals_adj | pct.1 | pct.2 |
|------|---------|---------------|-----------|-------|-------|
| RHO | 感光细胞 | 4.2 | 1.2e-50 | 0.85 | 0.03 |
| CRX | 感光细胞 | 2.8 | 3.4e-30 | 0.92 | 0.12 |
| VSX2 | 双极细胞 | 3.1 | 5.6e-40 | 0.78 | 0.05 |

**阅读步骤（四个重要的数值）：**

1. **logfoldchanges（log2 倍数变化）：** 这个基因在目标 cluster 里的平均表达量是其他细胞的多少倍。log2(4.2) ≈ 2⁴·² ≈ 18 倍。值越大越好。但要注意——如果一个基因在目标 cluster 里只有 1 个细胞异常高表达，logFC 也可能很高（所以不能只看它）
2. **pvals_adj（校正后的 p 值）：** 越小越好。10⁻⁵⁰ 的 p 值意味着"这个差异不太可能是随机产生的"。Fuxi 自动做了 Bonferroni 或 Benjamini-Hochberg 多重检验校正（multiple testing correction）——因为你在同时测试 20,000 个基因，需要控制假阳性率
3. **pct.1：** 该基因在这个 cluster 中有百分之多少的细胞检测到表达。pct.1 = 0.85 意味着 85% 的感光细胞表达了 RHO
4. **pct.2：** 该基因在所有其他细胞中有百分之多少的细胞检测到表达。pct.2 = 0.03 意味着其他细胞只有 3% 表达了 RHO

**关键判别规则：**

- 一个好标记：**高 logFC + 高 pct.1 + 低 pct.2**。RHO 在感光细胞中几乎完美符合
- 问题基因：**高 logFC + 低 pct.1**。意味着只在 1-2 个细胞中极高表达，其他细胞不表达——这可能是 RNA 捕获噪声或异常细胞
- 广泛表达基因（likely 不是好标记）：**高 pct.1 + 高 pct.2**。说明很多细胞类型都表达这个基因，不是鉴定细胞类型的好标记
- 非常罕见基因：**低 pct.1 + 极低 pct.2**（如 0.02 vs 0.001）——可能是某个亚群的特异性标记，但需要更多细胞验证

### 怎么看 dotplot

Fuxi 会自动生成 dotplot——这是单细胞分析中最常用的可视化工具之一。

Dotplot 同时编码了**三个维度的信息**：

- **X 轴：** 细胞类型（或 cluster）
- **Y 轴：** 基因
- **点的大小：** 该基因在该细胞类型中表达的细胞比例（对应表格中的 pct.1）
- **点的颜色：** 该基因在该细胞类型中的平均表达水平（通常经过 Z-score 归一化，红色 = 高，蓝色 = 低）

**读图技巧：**

- 先看点的**大小**。一个"干净"的标记基因应该是一个大大的圆点在目标 cell type 上，其他 cell type 上是极小的点或者没有点
- 再看点的**颜色**。在同个目标 cell type 上，颜色越红越好
- **注意"灰点"陷阱**：如果 dotplot 上某个基因几乎所有 cell type 都有中等大小的点（pct.1 ≈ 0.3-0.5），那它不是一个好的标记——即使它的颜色偏红，也只是因为所有细胞都在平均程度上表达

### 怎么看热图（heatmap）

热图和 dotplot 展示的信息类似，但编码方式不同：

- 每一行是一个基因，每一列是一个细胞（或求平均后的细胞类型）
- 颜色代表表达水平：红色 = 高表达，蓝色 = 低表达
- 通常在行和列上附加**层级聚类树（dendrogram）**，把相似的基因和相似的细胞聚类在一起

**读图技巧：**

- 如果热图的行和列形成了清晰的"方块"结构（block structure）——你的标记基因在区分细胞类型上效果很好
- 如果行基因在多个不相干的 cell type 上都表现为红色——这些基因不是特异性的标记
- 注意颜色条（color bar）的刻度——如果 Fuxi 用 Z-score（每行基因标准化到均值为 0），那么红色只表示"相对较高"，不表示"绝对高"

## 常见误区

**"标记基因 p 值最小=最好"**——不，p 值小只代表"统计上显著"，不代表"生物学上有意义"。因为单细胞数据细胞数多（数千到数万个细胞），一个非常小的 logFC（比如 1.2 倍）在数万个细胞面前可以产生极小的 p 值。永远**先看效应大小（logFC），再看 p 值**。

**"logFC 越大越好"**——不一定，还要看 pct.1。如果一个基因在 target cluster 里只有 2% 的细胞表达但 logFC = 6，那它可能是这 2% 的细胞单独形成了极高表达的异常信号。这通常意味着它在这个 cluster 里的表达集中在少数细胞上，不适合作为整群标记。

**"pct.2 = 0 好，说明其他细胞完全不表达"**——0 在单细胞数据中可能意味着"没有细胞达到检测阈值"，不一定意味着"这个基因不在其他细胞类型里存在"。可能只是表达量太低，低于检测线（dropout 噪声，见[第一章](01_中心法则与单细胞测序.md)）。

**"所有人都在用 Wilcoxon，它一定是万能的"**——Wilcoxon 确实是单细胞领域的标准方法，但它有一个重要的局限：**它只看排名，不看倍数**。两个群表达量差异极小的基因也可能因为排名一致低而 p 值极小，但生物上这种差异可能是无意义的。所以在阅读结果时，请跟着"先看 logFC，再看 pct.1，最后看 p_val"的顺序操作。

**"DE 分析做完，基因变化的生物学故事就自动出来了"**——不，DE 的结果只是一个"提名名单"。"感光细胞在疾病中有 500 个基因上调了"是一个观察，不是结论。你需要通过[富集分析](13_富集分析.md)把这些基因放进通路的上下文中，才能讲出生物学故事。

## 小结

- **标记基因 ≠ 差异表达基因**。标记基因是"一 vs 全体"用来鉴定细胞身份，差异表达是"条件 A vs B"用来发现生物学差异
- Wilcoxon 秩和检验是单细胞 DE 的标准方法：不看具体数值，看排名，稳健、不需要正态分布假设
- 读标记基因表关注四个值：**logFC > pct.1 > pvals_adj > pct.2**——按这个顺序读
- Dotplot 同时编码三个维度（细胞比例、表达水平、基因名称），是单细胞最基础的可视化工具
- Fuxi 的三层 DE 策略：标记基因（one-vs-rest）→ 配对时间点 DE（pairwise）→ 时间趋势基因（temporal），覆盖不同的生物学问题
