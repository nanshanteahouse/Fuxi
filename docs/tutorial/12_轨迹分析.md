# 轨迹分析

> 聚类是给细胞拍合影（每个细胞属于一个离散的组），轨迹分析是给细胞拍电影（细胞沿着一条连续的发育路径前进）。两者各有用途，不能互相替代。

## 我们为什么要做轨迹分析？

当你完成聚类（[Step 04](08_降维与聚类.md)）和注释（[Step 05](09_细胞类型注释.md)）之后，你已经有了一张"细胞类型地图"。地图上不同的点代表不同的细胞类型——感光细胞在这里、双极细胞在那里、Müller 胶质细胞在另一个地方。

但这里有一个问题：**这些细胞类型之间有关系吗？**

想象一个发育中的视网膜：一个视网膜前体细胞（retinal progenitor cell）先分裂、然后逐渐决定自己变成感光细胞还是双极细胞、最后成熟为功能性的细胞类型。这个过程是**连续的**——细胞不是瞬间从"前体"跳到"感光细胞"的。中间有大量过渡状态：

```
前体细胞 → 正在分化的细胞 → 几乎变成感光细胞 → 成熟的感光细胞
```

聚类把所有细胞按相似性分到不同的桶里；但"相似的细胞"可能只是处于不同过渡阶段的同一条分化路径。**轨迹分析（trajectory analysis）重建这条连续的路径。**

### 什么时候轨迹分析有意义？

- **发育时间序列：** 你有多个发育时间点的数据（胚胎期 → 出生 → 成年），想知道一条细胞谱系（lineage）是如何建立的。轨迹分析能把离散的时间点串联成连续的发育路径
- **稳态更新：** 即使不在发育中，很多组织也有"干细胞更新"的过程，比如肠道上皮的 Lgr5+ 干细胞持续产生分化细胞。轨迹分析可以用"伪时间"（pseudotime）重建这条更新路径
- **疾病进展：** 如果你有正常 → 早期病变 → 中晚期病变 → 肿瘤的样本，轨迹分析可以重建疾病进展路径

### 什么时候轨迹分析没有意义？

- **所有细胞都是终末分化（terminally differentiated）的成年细胞：** 如果你的数据只包含不同谱系的终末细胞（比如肝脏细胞 + 神经元 + 皮肤表皮细胞），它们之间没有连续的发育关系，轨迹分析会产生"强行连接"——把不相关的细胞类型连在一起，看起来有一条路径，但那只是算法在填入缺失数据
- **细胞数太少：** 轨迹推断需要足够的中间态细胞来"填满"路径。如果只有 100 个细胞，轨迹就是牵强附会
- **数据质量差：** 如果细胞在 UMAP 上分布散乱、没有清晰的子结构（见[Step 02 质量控制](06_质量控制.md)），轨迹分析的结果基本不可信

## 背后的原理

### PAGA——告诉你哪些细胞类型真正相连

**PAGA（Partition-based Graph Abstraction，基于分区的图抽象）** 是轨迹分析的起点，也是 Fuxi 采用的第一个方法。

直觉上，PAGA 做这件事：假设你有五个 cluster（A、B、C、D、E）。从 UMAP 上看，A 附近是 B、B 附近是 C……但 UMAP 的二维投影是有失真的——有时两个 cluster 在 UMAP 上看起来"挨着"，实际在高维空间里被一个真正的间隙（gap）隔开。

PAGA 在高维空间里问："cluster A 和 cluster B 之间的细胞密度是连续的（意味着它们可能是同一分化路径上的不同阶段），还是有一个显著的间隙（意味着它们是不同谱系的独立分支）？"

**PAGA 的输出是一个图**——节点是 cluster，边的粗细表示两个 cluster 之间连接的"可信度"。细线 = 弱连接、可能不应该连在一起；粗线 = 强连接、很可能是同一分化路径。

PAGA 的独特优势在于：它**不强迫把所有节点连成一个树**。如果数据中有两条独立的分化路径（比如视网膜前体细胞分化为感光细胞是一条路径，分化为双极细胞是另一条），PAGA 会自然地画出两条不相连的路径——而不是强行把它们绑在一起。这一点对复杂组织（尤其是胚胎发育数据）特别重要。

### DPT（扩散伪时间）——给细胞排序

PAGA 告诉你了"哪些细胞类型是连在一起的"。接下来你想知道："沿着这条路径，细胞是从 A→B→C 的方向发展的，还是 C→B→A？"这就引入了 DP T （diffusion pseudotime，扩散伪时间）。

想象你把一滴墨水（染色剂）滴入一杯平静的水中。墨水扩散——从滴入点开始，向外扩散。在扩散的前沿，墨水浓度最低；在滴入点，墨水浓度最高。

DPT 就是做这样的事：

1. 你告诉算法"这个细胞是起点"（root cell）——相当于"滴入墨水的地方"
2. DPT 从起点细胞开始，模拟一个"扩散过程"（数学上是构建马尔可夫链的扩散算子），计算每个细胞与起点之间的"扩散距离"
3. 离起点越远的细胞，DPT 值越大——即"伪时间（pseudotime）"越大
4. 最终每个细胞被赋予一个数值（0 ~ 约 100），代表"从起点出发沿着分化路径走了多远"

**伪时间（pseudotime）的直觉：** 因为你不能真正追踪每个细胞在时间中的变化（那需要活细胞成像，单细胞测序一次杀死细胞），所以你用"不同细胞处于同一路径的不同阶段"来近似时间。伪时间是空间上的"进度条"，不是时钟上的时间。

### 为什么起点选择这么重要？

如果你告诉算法"成年感光细胞是起点"，DPT 会从感光细胞向外扩散，最终告诉你——所有细胞"走"向双极细胞。这在数学上成立，但在生物学上颠倒黑白的。

起点选择是轨迹分析中最主观但最关键的一步。起点错了，整条轨迹的方向就反了。

Fuxi 的 root cell 自动检测逻辑（当你不手动指定时）：
1. 检查配置中是否设置了 root 细胞类型（`CFG.root_cell_type`，如 "Retinal Progenitor"）
2. 如果没有指定，查找该细胞类型的 marker 基因是否已知为"前体/祖细胞"标记（如 SOX2、PAX6）
3. 如果仍然不确定，以最早发育时间点（如 E14）的细胞作为默认起点
4. 如果没有任何发育时间信息，取数据集中排序第一的细胞作为 fallback

> **强烈建议**手动指定 root cell type——特别是当你了解自己的生物学系统时。自动检测逻辑只是保底方案，不是推荐方案。

### 分支（branching）

真实的发育过程通常是多叉的：一个前体细胞可以变成感光细胞、也可以变成双极细胞、也可以变成 Müller 胶质细胞。在轨迹上，这表现为一个**分支点（branch point）**——路径在此分裂为两条或多条子路径。

分支点对应的生物学含义是**细胞命运决定事件（cell fate decision）**：在这个点附近的细胞正在"决定"自己要变成什么。找到分支点的特异性基因，就找到了可能调控细胞命运决定的候选分子（branch-specific genes）。

## Fuxi 是怎么做的？

Fuxi 使用 **PAGA + DPT** 的串联策略，这也是 Scanpy 生态系统中最成熟的轨迹分析流程：

1. **计算 PAGA：** 基于 Step 04 的 Leiden 聚类结果，计算每个 cluster 对之间的连接可信度。这一步不需要任何用户输入
2. **计算 DPT：** 基于 PAGA 图，从 root cell 出发扩散，计算每个细胞的伪时间值
3. **分支检测：** DPT 在分支点处会自动分配"分支分组"（branch group）。如果检测到分支，你会得到分支 A 和分支 B 的分组信息
4. **分支特异性基因：** 比较不同分支之间的基因表达差异，找出在分支点附近高表达的基因（可能是调节细胞命运的转录因子）
5. **可视化：**
   - PAGA 图（节点为 cluster，边为连接可信度）
   - UMAP 上着色的伪时间梯度（从蓝色到黄色/红色）
   - 基因表达量沿伪时间的变化曲线

> 具体参数配置见 [Pipeline 指南](../pipeline_guide_zh-CN.md#step-08-轨迹分析)

## 我怎么看结果？

### PAGA 图

**第一眼应该看什么？**

- **哪些 cluster 之间有粗线（强连接）：** 这暗示它们是同一分化路径上的不同阶段。例如，如果"感光细胞前体"和"成熟感光细胞"之间有粗线，这条路径很可靠
- **哪些 cluster 之间没有连线（或极细线）：** 说明两个 cluster 被一个真正的间隙隔开，它们大概率不是同一分化路径上的。例如"感光细胞"和"Müller 胶质"之间通常没有连线——因为它们是两条独立的分化路径
- **图的结构：** 是"直线"（A-B-C-D，单一的线性分化路径）还是"树杈"（A 分支为 B 和 C）？"树杈"结构意味着有细胞命运分支事件

### UMAP 上的伪时间着色

Fuxi 会在 UMAP 上根据 DPT 值给每个细胞着色。颜色从起点处的深蓝色变为终末细胞处的亮黄色。

**重点看：**

- 伪时间梯度是否**平滑过渡**：如果颜色在地图上是一个从蓝到黄的渐变色（而不是斑驳的斑点），轨迹是合理的
- 伪时间方向是否**生物合理**：起点应该在"未分化"区域（如前体细胞区域），终末应该在"分化"区域
- **"跳跃"的异常区域**：如果一个区域的细胞伪时间值非常局部地突然跳到"高"或"低"——这可能是 UMAP 局部扭曲造成的伪迹，需要加一个问号

### 伪时间曲线图

Fuxi 会为每一个被标记为"可能重要的"基因画一张图：X 轴是伪时间，Y 轴是基因表达量。曲线是用平滑方法（如局部加权回归，LOESS）拟合的趋势。

**重点看：**

- **早期基因（early genes）：** 伪时间早期高表达，然后逐渐下降——通常是"前体标记"（如 SOX2、NOTCH1）
- **中期基因（intermediate genes）：** 伪时间中期出现一个峰——通常在"命运决定"窗口高表达，这些是**最关键的一类基因**，可能涉及细胞命运决定
- **晚期基因（late genes）：** 伪时间末期才逐渐升高——通常是"成熟标记"（如 RHO 对于感光细胞）
- **表达模式是否漂亮：** 一条平滑的上升或下降曲线比一个随机噪声模式更可信

### 如何验证一条轨迹的可信度？

1. **已知标记检验（marker validation）：** 找一个你知道应该在发育早期表达的标记（如 PAX6）——它的表达量应该在伪时间的早期高、晚期低。再找一个应该在成熟细胞中表达的标记（如 RHO）——它的表达应该在晚期高。如果两个都符合，你的轨迹方向是对的
2. **不同起点的一致性检验：** 如果你尝试用不同的 root cell（比如用 cluster A 当起点 vs. 用 cluster B 当起点），两条轨迹的相对排序是否一致？如果 A→B 和 B→A 给出截然不同的生物学故事，可能是你的数据不适合做轨迹分析
3. **外部验证：** 如果你的伪时间排序和已知的发育时间点（比如 E14、E16、P0）高度一致——大体的发育次序是对的。如果次序完全颠倒或者混乱的——轨迹不可信赖

## 常见误区

**"伪时间 = 真实时间"**——最危险的误解。伪时间是算法从横截面数据（不同细胞在同一时间点采集的"快照"）中推断出来的"进度"值，不是你直接测量的时间。两个细胞的 DPT 值差一倍，不代表后一个细胞比前一个细胞晚发育了"1 个单位的时间"。伪时间只有**相对排序**有意义，绝对值没有物理意义。

**"轨迹分析自动找到细胞命运决定基因"**——不。轨迹分析找到的是在分支点附近差异表达的基因。这些基因**可能**参与细胞命运决定，但同样可能是"路过"的伴随变化（乘客效应，passenger effect）。真正的命运决定因子需要通过功能实验验证（knockout、knockdown）。

**"PAGA 没有连接 = 两个细胞类型绝对没有关系"**——不一定。PAGA 在细胞数不足时可能漏掉真实连接。如果一个中间态细胞类型在你的数据中频率极低（比如 0.1%），PAGA 可能检测不到连接，因为中间态细胞太少，不足以在高维空间中构成"桥"。这是为什么 subcluster 或 sub-sampling 可以发现一些 PAGA 漏掉的结构。

**"所有数据都应该做轨迹分析"**——不。如果你的细胞都是终末分化的成年细胞，且它们来自不同的细胞谱系，轨迹分析强行连接无关的细胞。这种情况下，PAGA 图可能显示所有 cluster 之间都有细线——这是算法在"猜"，不是生物学事实。

**"轨迹越复杂 (=分支越多) 越有深度"**——不，通常一个简单的线性轨迹（1-2 个分支）更可信。复杂的多级分支往往是因为数据中存在过大的技术噪音。如果一个轨迹有 5 个以上的分支，你应该先回去检查[质量控制](06_质量控制.md)和[聚类参数](08_降维与聚类.md)。

## 小结

- 轨迹分析回答"细胞类型之间有什么连续关系"——它把聚类的"快照"变成发育的"电影"
- **PAGA** 判断哪些 cluster 真正连接（不是伪连接），**DPT** 给细胞排序（从起点到终末）
- 起点选择（root cell）是轨迹分析中主观性最强的步骤——选错了方向就是反的
- 轨迹分支是细胞命运决定的场所→分支特异性基因是命运决定因子的候选
- 轨迹分析结果需要用已知标记检验：早期标记应该出现在伪时间的早期，成熟标记出现在晚期
- **轨迹方向正确不代表绝对值有意义**——伪时间只有相对顺序可解释，不是真实时间
