# 如何配置新数据集

> 假设你已经运行过预处理脚本（`core/preprocess/preprocessor.py`），得到了 `dataset.yaml` 和 `config_{GSE}.py` 两个文件。本文档聚焦配置过程中需要做的**概念性决策**，而非每行参数的机械解释。

> 具体参数和配置语法见 [Pipeline 使用指南](../pipeline_guide_zh-CN.md#9-配置文件详解)。

## 你需要在开始前决定什么

### 1. 你的 `expression_type` 是什么？

这是第一个、也是最重要的决策。`expression_type` 告诉 Fuxi 你输入的计数矩阵是什么类型的数据：

| `expression_type` | 含义 | 典型场景 |
|-------------------|------|---------|
| `raw_counts` | 原始整数计数，未归一化 | 10X Cell Ranger 标准输出 |
| `TPM` | Transcripts Per Million，长度归一化 | 来自某些比对工具的预归一化输出 |
| `FPKM` / `RPKM` | Fragments Per Kilobase per Million | 早期 RNA-seq 常用格式，单细胞中少见 |
| `CPM` | Counts Per Million，仅深度归一化 | 某些平台的预处理输出 |
| `log2_scale` | 已经做了 log2 归一化 | 用户从其他渠道拿了已预处理的 h5ad |

**为什么这很重要？** Fuxi 的后续步骤（特别是 Scrublet 双细胞检测和标准化）对数据分布有假设。如果告诉管线"这是原始计数"，但实际上输入的是 TPM，Scrublet 可能会产生错误的双细胞评分。

**规则：** 输入是什么就设什么。如果不确定，查阅你的 `dataset.yaml`——预处理脚本在生成配置时会运行格式检测器，它会尽可能自动判定 `expression_type`。

### 2. 多样本还是单样本？

如果你的配置中有多于一个样本（这在公共数据库中很常见），你需要：

- 确保 `sample_map` 正确地将 barcode 后缀映射到样本名称
- 决定是否做批次校正（对于多样本通常需要）
- 考虑样本间是否有时间/处理关系（这会影响 `stage_map`）

**单样本很简单，但"单样本"不意味着只有一个样本。** 你可能有三个来自同一条件下的生物学重复——这是多个样本，通常也需要批次校正。

### 3. 是否有发育/时间序列？——为什么 `stage_map` 最容易配错

`stage_map` 不是 `sample_map`。它们的区别是：

- **`sample_map`**：告诉 Fuxi "这个 barcode 后缀对应哪个样本"，用于批次校正
- **`stage_map`**：告诉 Fuxi "这个样本属于哪个时间点/处理阶段"，用于时间序列 DE 和轨迹分析

最常见误区是：**在有多个样本的情况下，忘记设置 `stage_map`。** 此时 Fuxi 仍然能跑完流程——但所有涉及时间信息的分析（阶段比较 DE、轨迹分析中的"根"选择）将使用默认值，可能得出误导性结论。

> **建议：** 即使你的实验没有明确的时间点，也可以将 `stage_map` 留空或设为 `None`。只有在你知道样本的发育阶段/处理时间时，才需要设置它。

### 4. 是否有现成的知识库（tissue KB）？

Fuxi 目前内置的 Knowledge Base 只覆盖了**视网膜**组织。如果你研究的是其他组织：

- **KB 模式不可用**（因为没有对应组织的 KB）
- **备选方案 1：AI 模式**——如果启用了 AI（需要 API Key），LLM 可以直接根据标记基因注释
- **备选方案 2：Score_genes 模式**——在配置文件中的 `marker_dict` 里手动输入已知标记基因

Fuxi 的注释引擎会自动选择最佳方案：如果 `tissue_kb` 设置且存在对应 KB 文件 -> 使用 KB 模式；否则如果 AI 开启 -> AI 模式；否则 -> Score_genes。

### 5. 是否要使用 AI 注释？

要使用 AI 注释需要满足以下条件：
1. 在 `config.py` 中设置 `CFG.ai.enabled = True` 和 `CFG.ai.ai_annotation = True`
2. 设置环境变量 `LLM_API_KEY`（OpenAI / DeepSeek / vLLM / Ollama）
3. 有可用的互联网连接（如果使用远程模型）

关于何时信任/不信任 AI 注释，请见[附录 D](D_AI注释系统详解.md)。

### 6. 是否要基于标准模型选择数据识别？

如果数据来源于非标准模型（非人类/非小鼠），请特别注意：
- 基因符号可能已由测序平台提供，也可能需要手动转换
- Fuxi 提供 `ortholog.py`（通过 Ensembl 转换为人类同源基因），但自动化的适用范围有限
- AI 注释可能默认为人类/小鼠知识，可能需要额外在 `extra_context` (AI注释时补充组织背景)

## 配置哲学

**不要试图一次性优化所有参数。**

Fuxi 的默认参数经过了大量数据集调试，对于绝大多数 case 都能给出合理结果。正确的迭代策略：

1. **第一次跑**：保持除上述五个基本决策外的所有默认参数
2. **检查结果**：主要看 QC 诊断图、UMAP 和注释质量
3. **迭代调整**：根据 Step 02 的 QC 结果调整 `min_genes`/`max_genes` 等 QC 阈值；根据 Step 04 的注释质量决定是否切换注释模式
4. **不要无故触碰**：`random_seed`、`n_pcs`、`n_neighbors`（除非你有充分理由）

## 快速检查清单

在运行 `run_pipeline.py` 之前，逐一检查：

- [ ] `expression_type` 是否与输入数据匹配？
- [ ] `sample_map` 是否覆盖了 barcode 中的所有后缀数字？
- [ ] 如果多样本，是否需要 `stage_map`？
- [ ] 是否为你的组织指定了有效的 `tissue_kb`（或使用其他注释方式）？
- [ ] 如果使用 AI，API Key 和环境变量是否正确设置？
- [ ] 物种是否为 `human` / `mouse` / 其他（如其他，考虑同源基因映射）？
- [ ] 数据目录路径是否正确，文件是否存在？
- [ ] `dataset.yaml` 是否描述了正确的文件格式和样本信息？

## 自己检查

> - 你下载了一个 GEO 数据集，预处理后生成了 config 文件。在运行 pipeline 之前，你至少需要确认哪三个概念性决策？
> - `sample_map` 和 `stage_map` 有什么区别？如果设错了它们，会对下游分析产生什么具体影响？
> - 什么情况下应该坚持跑完第一次再调整参数？什么情况下应该在跑之前就先修改默认值？
