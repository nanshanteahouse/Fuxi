# 双细胞检测

> 第 01 步：找出那些"一个液滴里意外装了两个细胞"的数据点，在它们生成虚假的"新细胞类型"之前清除掉。

## 我们要解决什么问题？

当你运行 10X Genomics 的微流控设备时，它的工作原理大致是这样的：

> 细胞悬浮液 + 含 barcode 的凝胶珠 + 油 → 形成油包水的微液滴 → 每个液滴理想情况下包含一个细胞和一个凝胶珠 → 裂解 → 反转录 → 文库构建

问题是：**这是一个物理过程，遵循泊松分布**。在最优条件下，如果你将细胞浓度控制得当，大约 80%~90% 的液滴是空的（没有细胞），~10% 含一个细胞，~1% 含两个或更多的细胞。

这 1% 就是**双细胞（doublet）**——一次捕获了两个细胞的液滴。

> **一个形象的类比：** 想象你用一个滴管往冰格里加水。大多数格子刚好一滴。有些格子你手抖了一下，滴了两滴进去。这些"两滴水"的格子就是双细胞。

## 为什么一个双细胞就能搞乱整个分析？

双细胞的危险在于它不是随机的噪音，而是**系统性的假信号**。

假设一个 T 细胞和一个巨噬细胞被同时捕获到了同一个液滴中。测序之后它们的 RNA 被混合在一起。在你的数据分析中，这个"细胞"会同时表达 T 细胞的标记基因（如 CD3D）和巨噬细胞的标记基因（如 CD14）。

如果你没有去除它们，聚类算法会怎么处理？

一个既有 T 细胞标记又有巨噬细胞标记的"伪细胞"，可能会：
- 自成一个簇——看起来像是一个全新的细胞类型
- 或者成为 T 细胞簇和巨噬细胞簇之间的"桥梁"——使得两个簇的分界模糊

**更麻烦的是不同细胞类型组合的双细胞：**
- T 细胞 × B 细胞双细胞 → 看起来像某种同时表达 T 和 B 标记的不知名细胞
- 两个上皮细胞的双细胞 → 看起来像一个高表达的上皮细胞——几乎无法检测
- 相同细胞类型的双细胞（同型双细胞，homotypic doublet）→ 难以与真实细胞区分，但通常也最无害

## Scrublet 是怎么做到的？

Scrublet 的核心思想非常聪明，一句话概括就是：

> **它人工合成双细胞，然后问：每个真实细胞看起来更像一个真实细胞，还是更像一个合成双细胞？**

具体步骤：

1. **从你的数据中随机抽取两对细胞的表达谱，加起来，生成一个人工双细胞。** 重复这个操作很多次，建立起一个"合成双细胞库"。

2. **对每个真实细胞，计算一个"双细胞得分"（doublet score）。** 这个分数基于它和合成双细胞库的"邻近程度"——如果某个真实细胞的表达谱与合成双细胞非常相似，它很可能就是一个真实双细胞。

3. **自动确定一个阈值。** 利用"预期双细胞率"（expected doublet rate），Scrublet 确定一个切割点——得分超过这个值的细胞被标记为双细胞。

4. **如果自动阈值失败（有时发生），Fuxi 回退到手动阈值：** 直接用预期双细胞率作阈值。

## 为什么 Fuxi 按样本分别跑 Scrublet？

不同样本有不同的双细胞率。一个样本可能上样浓度高了，双细胞率是 3%；另一个样本可能更低，是 0.8%。如果全数据合并跑 Scrublet，高双细胞的样本会污染全局的得分分布。

**Fuxi 的分组运行逻辑：**

- 首先看配置文件中的 `scrublet_batch_key`，用该列分组
- 如果该列不存在，回退到 `sample` 或 `stage` 列
- 如果都不存在，在所有数据上整体运行一次

对于大规模样本（>15,000 个细胞），Fuxi 串行处理以避免内存溢出；小样本并行处理以提高速度。

## 预期双细胞率是怎么确定的？

10X 官方文档给出了一张回收细胞数与预期双细胞率的对应表。Fuxi 对这些数据点做线性回归，得到了一个连续拟合公式：

```
expected_doublet_rate (%) = 0.00000759 × 细胞数 + 0.00000053
```

这个公式的直觉是：你加载的细胞越多，每个液滴捕获多个细胞的概率就越高。Fuxi 会自动计算，并将结果钳制在 0.4%~15% 的安全范围内。

你当然也可以在配置中显式设置 `CFG.scrublet_expected_doublet_rate = 0.05`，这在你已知该组织类型的典型双细胞率时非常有用。

## Fuxi 的容错机制

单细胞数据很不规整。偶尔会出现某个样本的数据格式异常、基因名不匹配、或者稀疏矩阵操作失败的情况。

Fuxi 的哲学是：**在 Scrublet 失败时，将所有细胞标记为非双细胞，继续执行管线。**

```python
except Exception as e:
    warnings.warn(f"Scrublet failed for {sample_name}: {e}")
    return np.zeros(adata_sub.n_obs), np.zeros(adata_sub.n_obs, dtype=bool)
```

这意味着你不会因为一个坏样本而丢掉整个分析。代价是——这个样本中可能有一些残余的双细胞未被检测。但"保留少量双细胞"几乎总是比"整条管线卡死"要好。

## 为什么 TPM/FPKM/CPM 数据会自动跳过双细胞检测

这是新手最容易踩的坑之一。

**Scrublet 的统计模型假设输入是原始 UMI 计数（raw counts）。** 因为它模拟双细胞的方式是把两个"真实细胞"的表达谱加起来——这个加法只有在原始计数空间才有意义。

如果你的数据已经被标准化为 TPM、FPKM、CPM 甚至 log1p（这是很多人喜欢用的，因为看起来更"干净"），那么相加两个细胞就是无意义的：

- TPM 已经做了文库大小归一化，相加两个 TPM 值不代表任何生物学过程
- log1p 后的数据相加更是完全没有意义——log(A) + log(B) ≠ log(A+B)

Fuxi 会在 `expression_type` 不是 `raw_counts` 时自动禁用 Scrublet，并记录一条警告：

```
Scrublet is designed for raw UMI counts.
expression_type='log1p' violates the negative-binomial assumption.
Disabling Scrublet.
```

## 我怎么看结果？

日志会输出每个样本的双细胞统计：

```
Sample WT_D0: 48 / 4758 doublets (1.0%) [expected_rate=0.0091]
Sample WT_D7: 35 / 3921 doublets (0.9%) [expected_rate=0.0080]
Sample KO_D0: 20 / 2174 doublets (0.9%) [expected_rate=0.0060]
Sample KO_D7: 15 / 1881 doublets (0.8%) [expected_rate=0.0058]
Total predicted doublets: 118 / 12734 (0.9%)
```

关注以下几点：
- 各样本的双细胞率是否基本一致？如果某个样本的双细胞率异常高（>5%），可能意味着该样本的上样浓度过高
- 总双细胞率是否在合理范围内（0.5%~2%）？如果非常低（<0.1%），检查是否因为表达式类型不是 raw_counts 而跳过了检测

输出的 `01_doublet.h5ad` 文件相比 `00_raw.h5ad`，在 `obs` 中多了两列：
- `doublet_scores`：每个细胞的双细胞分数（0~1 之间的连续值）
- `predicted_doublet`：布尔值，True 表示被判定为双细胞

## 常见误区

**误区 1："双细胞检测和细胞过滤是同一件事。"**
不完全是。Step 01 只做检测（打标签），不执行过滤。数据集中仍然包含双细胞。真正的过滤发生在 [Step 02 QC](06_质量控制.md) 阶段——届时这些标签会被用来排除双细胞。这给了一个机会在过滤前检查数据。

**误区 2："双细胞处理完了就万事大吉，Step 02 只会看到健康的细胞。"**
Scrublet 不是完美的。同型双细胞（两个同类型细胞的双细胞）几乎无法检测——它们看起来就是一个高表达的细胞。另外，多细胞（三细胞、四细胞）的占比虽然极低，但也无法完全排除。始终保持一定的警觉心，尤其是在看到某个簇看起来异常"好"的时候。

**误区 3："这个样本只有 100 个细胞，预期双细胞率 0.4%，所以没有双细胞。"**
即使在极低的预期双细胞率下，小样本仍然可能碰巧有双细胞——只是概率低。Scrublet 仍然会运行并尝试检测。你不能保证 0 个双细胞。

**误区 4："我可以用 UMAP 或聚类结果来判断哪些细胞是双细胞。"**
不推荐。双细胞在 UMAP 上确实有时会形成一个独特的"中间区域"（在两个真实簇之间），但很多双细胞会完美嵌入到某个簇内部，肉眼完全无法分辨。**Scrublet 基于模拟+统计的方法，比人工观察 UMAP 可靠得多。**

## 小结

双细胞检测是单细胞分析中**最容易忽视但后果最严重**的一个预处理步骤。1% 的双细胞比例虽然不高，但在一个包含 20 个簇的数据集中，一个双细胞就可能产生一个伪簇。

Scrublet 用"模拟双细胞 + 比较真实细胞与模拟双细胞的相似性"的方式，巧妙地解决了这个问题。Fuxi 的按样本运行、自适应预期双细胞率、和容错机制，让这一步骤在大多数数据集上自动且稳定地工作。

如果你使用的是非计数数据（TPM/FPKM/CPM），这一步骤会被自动跳过。对于这种情况，请注意下游结果中可能出现一些难以解释的小簇——它们可能就是被你遗漏的双细胞。
