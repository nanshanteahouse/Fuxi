# 质量控制

> 第 02 步：用线粒体比例、基因检出数、文库复杂度等指标，找出并去除死细胞、空液滴和低质量细胞。

## 我们要解决什么问题？

你的数据已经排除了双细胞，但剩下的细胞并非个个都质量良好。10X 实验中不可避免地会捕获到：
- **死细胞或濒死细胞**：细胞膜破裂，mRNA 泄漏，只有线粒体 RNA 还留着
- **空液滴（empty droplet）**：只有溶液中的游离 RNA，没有细胞
- **碎片（debris）**：细胞碎片被错误地当成了细胞

这些低质量细胞如果不去除，后果和双细胞类似——它们会形成自己独特的簇（通常是"高线粒体、低基因数"的簇），扰乱下游分析。

## 四个关键 QC 指标的生物学含义

Fuxi 主要使用四个指标来做质量控制。理解它们的生物学意义，比记住阈值重要得多。

### 1. n_genes_by_counts（基因检出数）：这个细胞有多少种不同的基因？

也叫做 `nFeature_RNA`。每个细胞通常表达 500~5000 个不同的基因。如果远低于这个范围，意味着：
- **空液滴**（几乎测不到基因）
- **死细胞**（mRNA 已降解）
- **技术失败**（比如破膜不充分）

如果远高于这个范围，可能是：
- **双细胞/多细胞未被 Scrublet 检测到**（参见[双细胞检测](05_双细胞检测.md)）
- **天然大细胞**（如巨噬细胞，因为吞噬功能而表达更多的基因种类）

> **关键认知：** 基因检出数的正常范围因细胞类型而异。中性粒细胞（neutrophil）本身表达的基因就很少，而血浆 B 细胞（plasma B cell）因为大量产生抗体，通常只有少量高频表达的基因。一刀切的阈值会无效。

### 2. total_counts（文库大小 / UMI 总数）：这个细胞捕获了多少个转录本？

也叫 `nCount_RNA`。这个指标的值受以下因素影响：
- **细胞类型**（大细胞比小细胞有更多的 mRNA）
- **捕获效率**（不同批次可能有 10 倍的效率差异）
- **测序深度**（同一文库，测序深度越高，检测到的 UMI 越多）

**重要：总计数本身不是质量指标**——它只描述"测到了多少"，而不是"测得好不好"。一个健康的巨噬细胞有很高的总计数，一个健康的中性粒细胞有较低的总计数，两者可能都很好。这就是为什么 Fuxi 对 `total_counts` 只在 raw counts 模式下设置上限（排除罕见的极端高值），对其他表达类型则完全不设置限制。

### 3. pct_counts_mito（线粒体百分比）：这是最重要的 QC 指标

线粒体百分比（`% Mito`）是鉴别濒死细胞的最佳指标。

为什么？当一个细胞开始死亡时：
1. 细胞膜破裂
2. 细胞质中的 mRNA 开始降解（因为暴露在细胞质 RNase 中）
3. 线粒体有自己的双层膜结构，内部 mRNA 受到保护，降解更慢
4. 结果是：**细胞质 mRNA 减少 → 线粒体 mRNA 的比例相对升高**

所以，高 `pct_mito` > 通常意味着这个细胞在测序时已经快要死了。典型的健康细胞 `pct_mito` 在 5%~15% 之间（因组织类型而异，代谢活跃的组织如肝脏可能更高）。

需要注意，在某些特殊情况下高 `pct_mito` 也可能是真实的生物学信号：
- **代谢非常活跃的细胞**（如肝细胞、心肌细胞）
- **细胞处于缺氧应激中**（可能是在组织解离过程中受损，而非体内死亡）

### 4. log_genes_per_umi（复杂度 / 复杂率）

计算公式是：`log10(基因数) / log10(总计数)`。

这个指标衡量的是：**这个细胞的转录组是"广谱低表达"还是"窄谱高表达"？**

- 高复杂度（接近 1.0）：细胞表达的基因很多，每个基因的表达量相对均衡——典型的健康细胞
- 低复杂度（远低于 0.8）：细胞只表达少数几个基因（比如血红细胞几乎只表达血红蛋白基因）——这可能是真实的，也可能是技术问题

Fuxi 只在 raw counts 模式下使用这个指标，因为 TPM/CPM/FPKM 已经标准化了文库大小，复杂性指标失去了解释力。

## MAD 自适应阈值：Fuxi 的做法为什么更好？

很多教程会让你手动设置阈值：

```python
min_genes = 200
max_mito = 20
```

问题是，这些阈值来自他们自己的数据集。不同实验室、不同组织、不同测序运行的数据分布完全不同。

Fuxi 提供两种模式：

### MAD 模式（默认推荐）

MAD（Median Absolute Deviation，中位绝对偏差）的思路是：**计算数据的典型范围，然后剔除远离这个范围的离群点。**

计算步骤：
1. 计算该指标的中位数（median）
2. 计算 MAD = median(|每个值 - 中位数|)
3. 阈值 = median ± N × MAD（N 默认是 3，可配置）

> MAD 相当于对离群值更鲁棒的标准差。标准差对极值很敏感——一个极端值会拉大标准差，导致它"包容"更多异常点。MAD 以中位数为中心，不受极值影响。

**对于 pct_mito，阈值最终还会受到全局最大值的钳制**：`min(MAD 阈值, CFG.max_pct_mito)`。这确保了即使某个数据集的线粒体分布整体偏高（如肝脏），也不会超过绝对安全上限（通常是 20%~25%）。

### 硬阈值模式（传统做法）

直接使用配置中定义的值（如 `min_genes=200`、`max_mito=20`）。当你对某个特定组织的典型数值范围非常确定时可以使用。

### 该用哪个？

| 情况 | 推荐模式 |
|:--|:--|
| 新数据集、第一次分析 | MAD（自适应，稳定性好） |
| 反复分析的同类数据 | 硬阈值（控制严格、可复现） |
| 数据质量较差 | MAD（硬阈值可能误杀太多细胞） |
| 论文复现 | 硬阈值（与原始分析保持一致） |

## 什么类型的细胞会被误杀？

这是一个需要正视的事实：**质量控制必然误杀一些真实细胞。** 因为每个 QC 指标都存在生物学的真实的个体差异。

容易因 QC 被误杀（但仍然健康）的细胞类型：

| 细胞类型 | 看起来很像低质量 | 实际上是因为... |
|:--|:--|:--|
| 中性粒细胞（Neutrophil） | 低 n_genes、低复杂度 | 这些细胞的天然转录组就非常"窄" |
| 血浆 B 细胞（Plasma B cell） | 低复杂度 | 大量表达免疫球蛋白基因，其他基因相对少 |
| 肝细胞（Hepatocyte） | 高 pct_mito | 代谢非常活跃，线粒体含量天然高 |
| 肥大细胞（Mast cell） | 高 pct_mito | 分泌颗粒中富含线粒体 |

这是可以接受的代价——你可能失去了这几个细胞，但换来了全数据集免受低质量细胞的系统性污染。在单细胞分析中，**宁缺毋滥**是一个合理的原则。

## Fuxi 是怎么做的？

Step 02 的流程如下：

1. **加载数据**：读取 `01_doublet.h5ad`
2. **计算 QC 指标**：
   - 通过 `sc.pp.calculate_qc_metrics()` 计算 n_genes、total_counts、pct_mito、pct_ribo
   - 手动计算 `log_genes_per_umi`
3. **确定阈值**：MAD 自适应 或 硬阈值
4. **绘制诊断图**（在过滤之前——这是关键，展示原始分布）
5. **执行过滤**：
   - 标记为 doublet 的细胞
   - 各 QC 指标超标的细胞
6. **基因过滤**：去除在极少数细胞中表达的基因（`min_cells` 默认很少，仅去除极端情况）

## 我怎么看结果？

### 日志输出

```
Mode: adaptive (MAD × 3.0, nCount MAD × 5.0)
  n_genes_by_counts: median=1562, MAD=532  →  (lo=0, hi=3158)  [adaptive]
  total_counts:       median=4820, MAD=2150 →  hi=15670  [adaptive ×5.0]
  pct_counts_mt:      median=4.82%, MAD=2.15% →  hi=11.27%  [adaptive]
  log_genes_per_umi:  median=0.8284, MAD=0.0452 →  lo=0.6928  [adaptive]
```

关注：
- 中位数和 MAD 值是否合理？如果中位数 n_genes 只有 500 左右（对于大多数组织而言偏低），检查样本质量
- pct_mito 的中位数：正常组织通常在 5%~10% 之间
- 过滤后损失了多少细胞？合理的损失率是 5%~15%，如果超过 30%，检查是否有批次质量问题

输出日志也会给出逐条过滤明细：

```
Filtering breakdown:
    n_genes < 0:           0 (0.0%)
    n_genes > 3158:      752 (5.9%)
    mito > 11.27%:       435 (3.4%)
    nCount > 15670:       39 (0.3%)
    complexity < 0.6928:   8 (0.1%)
    Total (dedup):      1186 (9.3%)
After QC filtering: 11430 cells
```

### 诊断图

Fuxi 生成三张图，保存在 `figures/02_qc/` 目录下：

1. **nFeature 分布直方图**：显示基因检出数的分布，红色虚线是 MAD 阈值
2. **nCount vs nFeature 散点图**：每个点是一个细胞，颜色表示线粒体%——这是最有信息量的一张图。你应该能看到一个大致正相关的点云，远离云团的散点就是可能的离群点
3. **pct_mito 分布直方图**：线粒体%的分布，阈值标在右侧——检查是否存在明显的"双峰"（可能意味着一个健康的群体和一个垂死的群体）

## 常见误区

**误区 1："QC 指标越严格越好。"**
不对。过严的 QC 可能去除 40% 的数据，留下一群"看起来很好"但失去了真实生物多样性的细胞。特别是如果你研究的是应激反应或代谢活跃的细胞类型，严苛的 QC 恰好去掉了你最关心的细胞。

**误区 2："我可以用 MAD 模式然后不管它。"**
MAD 模式比硬阈值好，但也不是全自动的。如果数据集整体质量很差（比如 pct_mito 中位数已经是 20%），MAD 计算出的上限可能是 35%~40%——这太高了。你仍然需要检查图形结果并可能手动调整 `max_pct_mito`。

**误区 3："线粒体百分比低 = 细胞质量好。"**
不一定。一个来自 10X 跑得不好的样本，所有细胞都捕获效率很低，可能导致每个细胞的总计数很低——这时候 pct_mito 可能仍然很低（因为没有测到任何东西）。需要结合其他指标一起看。

**误区 4："使用了非 raw_counts 数据，QC 效果仍然和 raw counts 一样好。"**
错误的。Fuxi 在非 raw_counts 模式下会跳过 total_counts 上限和复杂度指标的过滤。因为这些指标在标准化后不再有物理意义。如果你使用的是 log1p 或 TPM 数据，QC 过滤的维度会比 raw counts 少，这是必要的妥协。

## 小结

QC 是单细胞分析中最重要的预防性步骤。四个指标各有侧重——基因检出数反映捕获到多少信息，总计数反映测序深度，线粒体百分比是最有力的"细胞健康指征"，复杂度则揭示表达谱的广度。

Fuxi 用 MAD 自适应阈值替代了一刀切的硬阈值，让过滤更合理、更稳健。但 QC 不是可以完全自动化的步骤——始终要查看诊断图，确认过滤效果符合直觉。

经过 QC 过滤后，剩下的就是高质量的单细胞数据。下一步，我们将对这个上万个基因 × 上万个细胞的矩阵进行[标准化与批次整合](07_标准化与批次整合.md)——让不同样本的数据可以"对齐"在一起分析。
