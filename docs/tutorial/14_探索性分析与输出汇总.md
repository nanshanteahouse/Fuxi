# 探索性分析与输出汇总

> 这是管线的"收包"步骤——把所有分析结果放在一个文件夹里，生成汇总报告，再帮你做最后一次"全局检查"：我的数据整体上是否合理？有没有什么被我遗漏的异常？

## 我们为什么要做这一步？

前面的每一步都生成了特定的输出：

- Step 04 → UMAP + cluster 标签
- Step 05 → 细胞类型注释
- Step 07 → 标记基因表格
- Step 09 → 富集分析气泡图

但问题在于：**这些输出散布在不同的子目录里。** 你要自己一个个打开文件夹翻看——很容易漏掉某个异常。

而且更重要的是：前面的每一步都是"局部优化"的——Step 04 只关心"哪些细胞聚在了一起"，Step 05 只关心"每个 cluster 应该叫什么名字"，Step 07 只关心"每个 cluster 的标记基因是什么"。

没有任何一步停下来问过全局问题：

- 我的注释从**整体**上看合理吗？（每一个 cluster 都有生物学上合理的标记吗？）
- 注释结果的**细胞组成**如何？不同样本之间、不同条件之间的细胞比例差异大吗？有生物学解释吗？
- 有没有某些 cluster 虽然被标注为"神经细胞"，但 QC 指标很差（线粒体比例高、基因数低）——说明这个"cluster"可能是低质量细胞的聚集，不是真实细胞类型？

**探索性分析（Exploratory Analysis）** 就是全景检查——不发现新东西，而是发现"异常东西"。

Fuxi 的 Step 10 做的也是这件事：汇总所有分析结果、生成整合的可视化报告、执行最后一次质量控制检查。方便你在进入生物学故事讲述之前，再做一次完整的"健康检查"。

> 注意：这不是"标准分析"的一部分，它是**包装和检验**的步骤。它的输出是整条管线在论文或报告中的"检查清单"。

## Fuxi 生成了什么（以及怎么看）

### 1. 细胞组成柱状图

**这是什么：** 一张柱状图，X轴 = 样本（或条件），Y轴 = 细胞比例，柱子的颜色分段表示不同细胞类型。

**怎么看：**

- **整体结构：** 一眼望去，每个样本的柱子颜色分布是否大致相似？如果样本 A 有 80% 双极细胞，样本 B 有 80% 感光细胞——你需要问：这是真实的生物学差异（比如两个样本来自眼睛的不同区域），还是批次效应未修正（[Step 03 整合](07_标准化与批次整合.md)）？
- **稀有细胞类型：** 某些细胞类型是否一直以低于 1% 的比例存在？如果一种细胞类型在所有样本中都是 < 0.1%，它可能是技术伪迹（少数低质量细胞被聚类成一个小群），需要标记一下
- **"缺失"的细胞类型：** 文献中报道视网膜有 5 类主要细胞，但你的数据中只看到了 3 类。是否因为某些细胞类型在你的样本中丢失了（组织消化过程 killed 它们）、还是你的注释分辨率不够高？这是一类很重要的问题
- **组间差异：** 如果你有两个条件（正常 vs 疾病），注意哪些细胞类型的比例在两组的对比中发生了变化——这是单细胞分析中最有信息量的问题

### 2. QC 指标在 UMAP 上的覆盖

**这是什么：** 三张 UMAP 图，每张图上用颜色表示一个 QC 指标——`n_genes`（每个细胞检测到的基因数）、`n_counts`（总 UMI 数）、`pct_mito`（线粒体基因比例）。

**怎么看：**

- **n_genes / n_counts 的低值区域：** UMAP 上有没有一片区域整体呈蓝色（低基因数）？如果这个区域刚好对应到某个你标注的细胞类型——**警惕**。可能是这个"细胞类型"实际上是一群低质量的细胞，被聚类在一起了
- **pct_mito 的高值区域：** 线粒体比例超过 20% 的区域应该引起注意（[Step 02 QC](06_质量控制.md) 中我们曾说线粒体比例高 = 细胞膜破损，RNA 泄漏）。如果一个 cluster 的 pct_mito 显著高于其他 cluster，它大概率是低质量细胞而不是真实细胞类型
- **"自洽性"检查：** 如果 n_genes 的梯度在 UMAP 上呈现"一片区域高、另一片低"的连续变化（而非离散的高低分离），这说明基因数（测序深度）差异可能是影响 UMAP 聚类结构的因素——你可能需要重新检查你的归一化是否充分（[Step 03](07_标准化与批次整合.md)）

### 3. 已知标记基因在 UMAP 上的表达

**这是什么：** 从你的配置文件中读取已配置的已知标记基因（`CFG.known_markers`），在 UMAP 上用颜色表示每个基因的表达水平。

**怎么看：**

- **验证注释：** 如果你标注了"感光细胞"，那么 RHO（视紫红质）的表达应该集中在这个 cluster 的区域。如果 RHO 在多个 cluster 里都有表达——你的注释可能混入了多种细胞类型，或者某些 cluster 其实是同一细胞类型的亚群
- **"缺失"的标记：** 一个已知的感光细胞标记在"感光细胞"区域中不表达？这有两种可能：（1）你的样本来源的物种不是人类，而这个标记是物种特异的；（2）你的"感光细胞"注释其实是错误。你需要额外验证
- **"意外的"表达：** 一个标记基因在毫不相干的 cluster 中高表达——可能是您的标记本身特异性不佳（许多"经典标记"在大量测序数据中其实宽度不窄），也可能是数据整合造成的假关联

### 4. Cluster 大小汇总

**这是什么：** 按 cluster 分组的细胞计数表格，列出每个 cluster 的细胞数量和比例。

**怎么看：**

- **极小的 cluster（< 30 个细胞）：** 这些几乎可以肯定是"离群值集群"——可能由 1-2 个异常细胞和它们的"近邻"在 UMAP 上形成的。通常情况下，这些 cluster 没有或没有生物学意义
- **单个 cluster 占比过大（> 50%）：** 如果一种细胞类型占所有细胞的比例超过 50%——你需要理解为什么。如果检索文献发现视网膜中双极细胞占所有细胞的比例大约是 40%……那就 OK。如果比例异常，可能原因包括：组织消化位置（某个细胞类型比其他类型更耐消化）、接种偏差、聚类参数过低
- **跨样本的 cluster 大小变化：** 同样 cluster 分配的细胞数在不同条件之间明显不同——差异表达的两种可能性：真实生物学变化 vs 技术差异（消化效率、测序深度）

## 从"检查"到"故事"

探索性分析的终极目的是问一个问题：**我的数据足够可靠，可以进入生物学故事了吗？**

"可靠"不等于"完美"。scRNA-seq 数据总有噪音、总有难以发现的异常。但不等于所有。你要做到的是排除那些可能导致**误导性结论**的重大问题。

**检查清单：**

- [ ] 每个注释的细胞类型都有至少 1-2 个已知标记基因的支持
- [ ] 没有 cluster 同时表现出"低 n_genes + 高 pct_mito"（低质量细胞的典型特征）
- [ ] 稀有细胞类型（< 1%）没有被当作独立生物学结论的基础
- [ ] 条件间细胞比例的变化可以用生物学解释，或者你知道这可能是技术差异
- [ ] UMAP 上没有"莫名其妙"的分离（比如所有样本 A 的细胞和所有样本 B 的细胞分开形成两块——说明批次效应可能没有完全校正）
- [ ] 极小 cluster（< 30 个细胞）没有被当作独立细胞类型进行分析

如果所有复选框通过——恭喜你，你有一个数据合理的基础，可以开始你最关心的生物学故事了。

## 常见误区

**"这一步是自动化的，我不需要看"**——完全错误。自动化 = Fuxi 帮你把图"画"了，不是 Fuxi 帮你"看"了。单细胞分析中，最容易犯的错误就是"信任图"而不"读图"。Step 10 生成了最多的可视化文件，也最容易因为"太多了不看"而被忽略。

**"如果所有 QC 贴图的颜色看起来健康，数据就是好的"**——不是这样。QC 指标颜色健康意味着"没有硬件级别的严重问题"，但不意味着"聚类和注释是正确的"。一个完全真实的细胞类型如果被错误的分配了错误的注释，QC 指标颜色看起来也正常——但生物学故事完全错了。

**"细胞组成比例的变化可以直接作为论文结论"**——不。细胞比例的差异受到大量技术因素的影响：组织消化速度（某些细胞类型更容易从组织中解离）、微流控捕获的优先性、细胞大小的影响等。如果你的差异条件看起来"完美"说明了你的假设，先做一个简单的验证，用不同的实验方法（如免疫荧光染色、flow cytometry 验证比例变化）确认这不是捕获偏差。

**"这张图不好看，所以数据有问题"**——UMAP 图上的某些 cluster"歪歪扭扭"而不是完美的圆形，这不代表数据有问题。真实的生物数据本来就是扭曲的——某些细胞类型的异质性比另一个大，所以它们在 UMAP 上天然就比另一个更大更分散。

## 小结

- Step 10 不是"发现新东西"的步骤，是"全面检查"的步骤——在看深入分析之前，你在这里要找出所有"全局异常"
- 四个主要输出：细胞组成图、QC 叠 UMAP、已知标记表达图、cluster 大小汇总
- 检查清单的核心问题是：有没有一个 cluster 同时满足（低基因数 + 高线粒体比例 + 无双标记基因 + 细胞数极少）？如果有，它大概率是无生物学意义的低质量细胞群，需要从后续分析中排除
- 通过全局检查后，你才能放心地进入生物学故事的讲述
