# 标准化与批次整合

> 第 03 步：消除文库大小差异和批次效应这两个最大的技术噪音，让生物学信号浮出水面。

## 我们要解决什么问题？

如果你现在打开 `02_qc.h5ad` 看看 `X` 矩阵——它是一个稀疏矩阵，里面的值是什么？**原始 UMI 计数**。Cell A: 20,000 个 UMI，Cell B: 2,000 个 UMI。

想象一下，基因 GAPDH 在 Cell A 中检测到了 200 个 UMI，在 Cell B 中检测到了 20 个 UMI。表面上看 GAPDH 在 Cell A 中表达显著更高——但实际上是 Cell A 的"总测量量"就是 Cell B 的 10 倍。这是技术差异，不是生物学差异。

标准化就是要消除这种"文库大小差异"。

但还有一个更大的问题：**批次效应（batch effect）**。

> **一个形象的类比：** 你用一个相机拍了一张风景照，你的朋友用另一个相机拍了同一片风景。两张照片白平衡不同、对比度不同、甚至色彩饱和度也不同。批次校正就像是把两张照片"色彩矫正"成同一台相机拍出来的效果。

在单细胞分析中，"同一片风景"就是同一个生物学样本，"不同相机"可能就是：
- 不同运行批次（今天做 vs 下周做）
- 不同试剂批次（Lot A vs Lot B）
- 不同测序通道（Lane 1 vs Lane 2）
- 甚至不同实验人员操作

这些差异会系统地影响表达测量，在聚类时使"相同样本的细胞"簇在一起，而非"相同生物学类型的细胞"簇在一起。

## 标准化 vs. 整合——有什么区别？

这是新手最容易混淆的两个概念：

| | 标准化（Normalization） | 整合/批次校正（Integration/Batch Correction） |
|:--|:--|:--|
| **解决什么问题** | 文库大小差异 | 批次效应 |
| **作用范围** | 同一个样本内 | 跨样本/跨实验 |
| **方法** | 除以总计数 × 缩放因子 + log | PCA + Harmony |
| **改变表达值吗？** | 改变 X 矩阵 | 不改变 X，改变降维坐标 |
| **可以单独做吗？** | 可以 | 不能——必须先标准化 |

简单说：标准化让每个样本内部可比，整合让样本之间可比。

## 为什么做 log1p 变换（而不是 log）？

基因表达数据大致遵循**对数正态分布（log-normal distribution）**。也就是说，如果你对表达值取对数，数据看起来比较接近正态分布——这对 PCA 等依赖正态假定的方法非常友好。

但直接做 `log(0)` 有问题——log(0) 是负无穷。所以单细胞领域使用 **log1p** = log(1 + x)：

- 当 x=0 时，log1p(0) = log(1) = 0 —— 漂亮地处理了零值
- 当 x 很大时，log1p(x) ≈ log(x) —— 与常规对数几乎没有差异

> **Fuxi 的具体做法：** 先 `normalize_total(target_sum=10000)`（把每个细胞的总计数缩放到 10,000），再 `log1p`。这个缩放因子 10,000 是一个惯例，相当于把每个细胞的"测量尺度"统一到同一个量级上。

## 什么是 pearson 残差？Fuxi 现在不是直接用 log-normalize 吗？

等一下——你可能注意到了，Fuxi 实际上使用的是隔夜版 normalize_total + log1p 的经典方案，而不是 Pearson 残差法。这是**故意的**。

Pearson 残差方法（SCTransform）的核心思想非常优雅：

> 传统方法 | 先除以总计数，再 log。隐含假设：计数和表达量成线性比例关系。
> Pearson 残差 | 对每个基因建立一个"期望模型"——给定这个基因的平均表达水平和这个细胞的总文库大小，期望观察到多少计数？然后问：观察到的值比期望值高还是低？

负残差 = 比预期表达得少；正残差 = 比预期表达得多。

这种方法的好处是它自然地处理了**均值和方差的关系**——在 UMI 数据中，高表达的基因方差更大，Pearson 残差自动做了"标准化"。但它的缺点是**残差不是表达量**——你不能说"残差 2.5 意味着比平均值高 2.5 倍"。

Fuxi 选择经典的 log-normalize 路径，是因为：
1. 它与大多数下游工具兼容
2. 解释直观（log1p 值有明确的物理量纲）
3. 对后续标记基因分析（Wilcoxon rank-sum test）影响小

**如果你的数据集是 TPM/FPKM/CPM**，则不需要 normalize_total（这些已经做了文库标准化），直接 log1p 即可。Fuxi 的 `expression_type` 配置项会自动处理这一区别。

## 高变基因（HVG）：为什么要丢掉 80% 的基因？

在主流的 scRNA-seq 分析中，经过标准化后，我们通常**只保留 2000~5000 个高变基因用于下游降维和聚类**。

为什么？两个原因：

1. **计算效率**：20,000 个基因 × 10,000 个细胞 = 2 亿个数据点。PCA 在 2 亿个点上做 SVD 分解非常慢。
2. **信号噪声比**：大部分基因在细胞之间的差异不大——许多是"看家基因"（在所有细胞中稳定表达），许多是"零膨胀"的（在几乎所有细胞中都不表达）。只有高变基因携带了区分细胞类型的信息。

> **HVG 选择方法：** Fuxi 按优先级尝试多种算法：`seurat_v3` → `cell_ranger` → `seurat` → 手动方差法。`seurat_v3` 通常是首选，因为它能更好地识别罕见细胞群的特异性 marker，而且支持 `batch_key` 参数——跨批次选择 HVG 时，可以避免某一批次的"批次特异性高表达基因"被选中，而另一批次的真实 HVG 被遗漏。

### 那丢掉的 80% 的基因去哪儿了？

全部保留在 `adata.raw` 中！这个 `.raw` 是 AnnData 中一个特殊的槽位，保存的是标准化 + log1p 后的**全基因矩阵**。

当你做标记基因分析（Step 07）时，`sc.tl.rank_genes_groups()` 会使用 `.raw` 中的数据，因为你需要搜索所有基因，不仅仅是 HVG。

## Regression Out：回归去除技术变异

在 HVG 子集上，Fuxi 会做一步 `sc.pp.regress_out()`，回归掉两个技术协变量：`total_counts` 和 `pct_counts_mito`。

这样做的好处是：去除**由测序深度和细胞健康度引起的系统性表达差异**。一个细胞测得更深，所有基因的表达都会偏高——regress_out 对这种"全局性偏移"做了一次"减法"。

但 regress_out 也会产生浮点数的精度变化（从 float32 → float64），所以 Fuxi 之后会强制降回 float32。

## PCA：从 4000 维到 100 维

即使只保留 HVG，我们仍然有 ~4000 个维度。在 4000 维空间中做聚类是没有意义的——这就是**维度灾难（curse of dimensionality）**：在高维空间中，所有点之间的距离都变得差不多大，距离的概念失去了意义。

PCA（主成分分析，Principal Component Analysis）的核心思路：**找到数据中方差最大的方向**。

- **PC1**：数据差异最大的轴——在所有细胞之间变化最大的"基因组合"
- **PC2**：与 PC1 正交的、方差第二大的轴
- **PC3**、**PC4**……依此类推

每个主成分是多个基因的线性组合。你不需要理解"这个 PC 对应的具体权重是什么"，只需要知道：**前 N 个 PC 已经抓住了数据中最主要的信息**。

> **值得注意的是：PC1 往往是文库大小轴。** 因为"细胞 A 比细胞 B 测得更深"是数据中方差最大的一种变化模式。这也在意料之中——标准化无法完全消除文库大小差异在高维空间的影响。这更加说明为什么你需要用足够多的 PC（Fuxi 默认 ~100），而不是只用前几个。

Fuxi 还会生成一张 **PCA elbow 图**（`figures/03_integrate/pca_elbow.png`），显示每个 PC 解释的方差比例。elbow（肘部）位置通常是你应该使用的 PC 数——但 Fuxi 的 `n_pcs_use` 是预设的，elbow 图是一个帮助验证的参考。

## Harmony：真正的批次校正

这是关键的一步。Harmony 在 PCA 空间中对细胞坐标进行校正，使得**来自不同批次但同一生物学状态的细胞在 PCA 空间中对齐**。

### Harmony 是怎么工作的？——极其简化的版本

1. **软聚类（soft clustering）**：在 PCA 空间中，Harmony 先做一次模糊聚类——每个细胞不是属于一个簇，而是属于所有簇的"概率组合"
2. **检查簇的组成**：如果一个簇主要由批次 A 的细胞构成，说明这个簇是"批次特异性的"
3. **调整**：Harmony 迭代地调整细胞坐标，使每个簇的批次组成更均匀
4. **重复**：直到收敛（通常是 10~20 次迭代）

### Harmony 对比图

Fuxi 会生成一张左右对比图：左边是原始 PCA（按批次着色），右边是 Harmony 校正后的 PCA（按批次着色）。如果校正有效，你应该看到：

- **左边**：相同批次的细胞聚在一起，不同批次明显分离
- **右边**：不同批次的细胞混在一起，不再按批次聚集

> ⚠️ **一个重要警告：Harmony 确实改变了数据。** 它的输出 `X_pca_harmony` 是校正后的坐标——下游的聚类、UMAP 都基于这个校正后的坐标。所以 Harmony 不是"不改变数据只改变可视化"，它确实会影响聚类结果。

如果 Harmony 因任何原因失败（如 batch_key 列缺失、数值异常），Fuxi 会 graceful fallback：将未校正的 PCA 坐标复制到 `X_pca_harmony` 中，管线继续运行。

## Fuxi 是怎么做的？（操作顺序总结）

```
02_qc.h5ad → HVG 选择 → 子集到 HVG → regress_out → 
normalize_total + log1p (HVG 子集) → normalize_total + log1p (全基因) → 
保存到 .raw → PCA → Harmony → 03_integrated.h5ad
```

关键设计决策：
- **先 regress_out，再 normalize**: 回归是在 raw 缩放后的数据上做（scale 敏感的），而非 log 后的数据
- **HVG 子集和全基因分别做 normalize**: HVG 子集的 X 用于 PCA 和聚类；全基因矩阵存 `.raw` 用于标记基因分析
- **两个矩阵分开归一化**: 避免了全基因矩阵的冗余计算

## 我怎么看结果？

### 日志关键指标

```
PCA (50 components)...
  top-5 variance ratio: 0.3142
  Cumulative variance ratio first 50 PCs: 0.8521
```

- 前 5 个 PC 解释了 30%~50% 的方差是正常的
- 如果前 5 个 PC 只解释了 5%，说明数据信噪比很低——可能是数据质量问题

```
HVG count: 4000
```

- 4000 是合理数字。如果只选了不到 500 个 HVG，可能因为：
  - 数据集非常小（~100 个细胞）
  - 基因表达矩阵的某些 HVG 方法失败了，回退到手动方差法

```
Harmony correction (batch_key=sample)...
  Harmony complete, output shape: (11430, 50)
```

- 输出的 PC 维度取决于 `n_pcs_use`

### Harmony 对比图

查看 `figures/03_integrate/harmony_comparison.png`：
- 校正后批次之间应该混在一起
- 如果校正后批次仍然分离，考虑增大 `harmony_max_iter`
- 如果图谱反而更乱了，可能是 batch_key 设置错误（比如把生物学分组当成批次去掉了）

## 常见误区

**误区 1："批次校正去掉了所有批次差异，所以聚类结果不受批次影响。"**
不完全正确。Harmony 尽力保留生物学变异的同时去除批次差异，但这是有代价的。如果某个样本的细胞类型是另一个样本没有的（例如疾病组的某种稀有细胞类型在对照组不存在），Harmony 可能错误地将这些细胞"拉向"其他类型。**批次校正不是魔术**——好的实验设计仍然是最重要的。

**误区 2："标准化后就可以比较不同基因的表达绝对值了。"**
不对。标准化只是让细胞之间某个基因的相对表达可以比较。但在同一细胞中，不同基因之间的表达绝对值仍然没有直接的物理可比性（因为不同基因的捕获效率不同）。

**误区 3："Log1p 变换会让低表达基因的差异变大，高表达基因的差异变小，这很危险。"**
这正是 log 变换的目的。一个基因的平均表达从 0.1 升到 0.2（变化 2 倍），在 UMI 水平可能只是从 5 个 UMI 到 10 个 UMI——容易被噪声淹没。log 变换把变化从"绝对差异"转变为"倍数变化"视角，这更符合生物学直觉。

**误区 4："不需要关注 HVG 选了什么基因。"**
有时值得关注。如果 HVG 列表中大量是线粒体基因、核糖体基因，说明这些"技术噪声"源仍然主导了数据中的主要变异——即使经过 regress_out 和标准化。这可能意味着这批数据中死细胞的比例很高。

**误区 5："我做的是单样本分析，不需要 Harmony。"**
正确。单样本分析不需要批次校正。`use_harmony` 设 False 即可。但标准化仍然是必需的。

## 小结

标准化与批次整合是整个管线中最概念密集的一步，也是技术性最强的一步。你需要理解：
- **标准化**解决"不同细胞测到了不同量的 RNA"的问题
- **log1p** 让数据更接近正态分布，适合下游 PCA
- **HVG 选择**去掉冗余维度，保留区分信号
- **PC1 往往是文库大小**——这是已知的、可以接受的技术瑕疵
- **Harmony** 对齐不同批次的细胞坐标，是跨样本分析的关键

完成这一步后，你的数据已经准备好进行真正的生物学分析——首先是用 [UMAP 和 Leiden 来做聚类与可视化](08_降维与聚类.md)。

## 自己检查

> - 标准化和整合解决的是不同的问题。标准化去掉了什么？整合去掉了什么？
> - 你的数据中有 4 个样本，来自两个不同的测序批次。UMAP 上每个样本自成一团。这时你应该检查什么参数？
> - Harmony 校正之后，数据到底变没变？如果有审稿人说"Harmony 只是改变了可视化"，你怎么回应？
